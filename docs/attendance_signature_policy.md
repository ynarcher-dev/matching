# QR 서명 체크인 기능 보류 결정

> 작성일: 2026-06-26  
> 상태: 기획 결정 메모. QR 기반 서명 체크인은 현재 범위에서 제외한다.

## 1. 결정

30초 만료 서명 QR을 이용한 현장 체크인 기능은 구현하지 않는다.

현재 운영 방식에서는 현장 관리자 또는 전문가가 직접 출석을 찍으면 충분하다고 판단한다.

## 2. 이유

- 이미 관리자 진행 현황 화면에서 전문가/스타트업 출석을 직접 처리할 수 있다.
- 전문가 화면에서도 본인 출석과 상담 시작 처리가 가능하다.
- QR 서명 체크인은 현장 UX와 검증 방식이 복잡해지지만, 현재 필수 운영 요구는 아니다.
- 무료 운영 전환과 현장 사진 업로드, 알림 API 설정이 더 높은 우선순위다.

## 3. 유지할 출석 방식

관리자/현장담당자:

- 행사 진행 현황에서 참가자 출석을 직접 변경
- 실수로 누른 경우 미정 상태로 복구
- 노쇼 처리 시 사유 입력

전문가:

- 본인 출석 처리
- 담당 스타트업 출석 확인 또는 처리
- 상담 시작 버튼으로 세션 상태 전환

## 4. 개발 영향

Phase 7 기존 항목 중 다음 작업은 제거 또는 보류 처리한다.

```text
30초 만료 서명 QR 및 현장 스태프 수동 출석 처리 구현
```

수동 출석 처리는 이미 구현된 진행 현황/전문가 화면을 사용한다. 별도 QR 서명 플로우는 만들지 않는다.

## 5. 추후 재검토 조건

다음 요구가 생기면 애드온으로 재검토한다.

- 참가자 본인 기기에서 직접 체크인이 반드시 필요함
- 출석 위변조 방지를 위해 단기 서명 QR이 필요함
- 대규모 행사에서 현장담당자 수동 처리로 병목이 발생함
- 외부 감사 목적으로 사용자 직접 서명/체크인 로그가 필요함
