# [기능 명세] 관리자 - 스타트업/전문가 DB 관리 및 CSV 일괄 등록 (Admin User Management)

본 문서는 스타트업과 전문가 데이터베이스를 조회, 개별 등록 및 CSV 파일을 통해 일괄 업로드 관리하는 화면의 상세 사양을 정의합니다.

---

## 1. 스타트업/전문가 DB 관리 화면 (User DB Management View)

## 1. 스타트업/전문가 DB 관리 화면 (Participant DB Management View)

### 1.1 UI/UX 개요
- **진입 경로**: 최고관리자 메뉴의 `스타트업 DB`(`/admin/startups`) 및 `전문가 DB`(`/admin/experts`) 라우트에서 각각 진입합니다.
- **구현 특징**: 단일 공통 뷰 컴포넌트인 `ParticipantDbView.tsx`를 공유하여 구현하되, 각 라우트 진입 시 `role` 파라미터를 통해 대상 역할을 고정하여 운영자가 완전히 별개의 독립된 DB 화면으로 인지하도록 구성하였습니다. (구 `/admin/users` 진입 시 `/admin/startups`로 자동 리다이렉트 처리)
- **주요 기능**: 등록된 모든 전문가와 스타트업의 신상 및 첨부파일 정보를 공통 `DataTable` 형태로 조회하고, 개별 추가/수정(`UserDetailModal`) 및 CSV 일괄 등록과 소프트 삭제를 관리합니다.

### 1.2 화면 구성 (Layout Spec)
1. **조작 및 툴바 영역 (`PageToolbar`)**:
   - `개별 추가` 버튼: 수동 등록 모달인 `UserDetailModal` 오픈.
   - `CSV 일괄 업로드` 버튼: 파서가 포함된 일괄 등록 UI 활성화.
   - `진단 정보(OTP)` 토글: 기본적으로 숨겨져 있는 세션 제어 및 OTP 로그 진단 영역 활성화.
   - 검색창 (`SearchInput`) 및 필터 칩스 (`FilterChips`).
2. **참가자 테이블 리스트 (`DataTable`)**:
   - **공통 컬럼**: 성명, 이메일, 연락처, 최근 로그인, 등록일.
   - **스타트업 전용 컬럼**: 기업명, 대표자명, 담당자명, 관심 분야 칩스, IR/소개서 (업로드 상태 배지 + 파일 보기 + 마지막 업로드 주체 및 시각 동시 표시).
   - **전문가 전용 컬럼**: 소속, 직책, 전문 분야 칩스, 프로필 (Signed URL을 통한 사진 이미지 확인 배지).
   - **조작 컬럼**: `수정` | `삭제` (목록 숨김 및 로그인 차단)
3. **페이지네이션 (`Pagination`)**:
   - 기본값 한 페이지당 30개씩 페이징 처리하여 데이터 로딩 성능 최적화.

### 1.3 진단 정보(OTP) 토글 및 고급 세션 제어
- 참가자의 기본 로그인 방식이 `이름 + 휴대전화번호`로 변경됨에 따라, 일상적인 DB 운영 상황에서 불필요한 OTP 관련 지표와 조작 실수를 방지하기 위해 **"진단 정보(OTP)" 토글** 제어를 도입했습니다.
- **토글 OFF (기본값)**: 일반적인 인적사항 조회 및 첨부파일 관리 기능에 집중하며, 테이블에서 OTP 정보와 복잡한 세션 제어 액션이 노출되지 않습니다.
- **토글 ON**: 현장에서 발생한 인증 장애 분석 및 개별 예외 지원을 위해 아래의 특수 컬럼 및 조작 기능이 테이블에 추가 노출됩니다.
  - `최근 OTP`: 최근 챌린지 생성 시각 및 발송 결과 로그.
  - `1회용 링크`: 현장에서 즉시 로그인할 수 있는 1회용 긴급 로그인 링크 발급 모달(`EmergencyLinkModal`) 오픈 버튼 활성화.
  - `세션 무효화`: 보안 사고 우려 시 해당 사용자의 기존 로그인 토큰 세션을 강제 무효화하는 버튼 활성화.
- **최근 로그인 추적**: 참가자가 일반 이름+휴대전화 로그인을 하거나, 관리자가 발급한 긴급 로그인 링크를 소비하여 시스템에 로그인할 때마다 `users.last_login_at` 컬럼이 서버 측에서 실시간 갱신되어 표시됩니다.
- **소개서 업로드 주체 추적**: 스타트업이 자율적으로 IR/소개서를 업로드하거나 관리자가 대행 업로드할 시, DB 트리거(`track_proposal_upload`)를 통해 마지막 업로드 주체(`proposal_uploaded_by` - 업로더 이름)와 업로드 시각(`proposal_uploaded_at`)이 자동으로 기록되어 테이블 내에 명시됩니다.

### 1.4 테이블 공통 기준
- 검색 대상: 이름, 기업명/소속, 이메일, 연락처. (검색어 입력 시 디바운스 적용)
- 정렬 기준: 기본 등록일 내림차순(최신 등록 유저 우선). 이름, 기업명/소속, 등록일, 최근 로그인 기준으로 안정 정렬(`sortRows`) 및 오름차순/내림차순 토글 지원.
- 필터 조건: IR/소개서 파일 첨부 유무, 프로필 사진 첨부 유무, 최근 로그인 유무, 현장 1회용 링크 활성 여부 등을 제공합니다.

## 2. CSV 일괄 업로드 파서 및 검증 기능 (CSV Bulk Upload)

### 2.1 파일 업로드 워크플로우
1. 관리자가 "CSV 일괄 업로드" 클릭 후 템플릿 양식에 맞춰 작성된 `.csv` 파일 선택.
2. 프론트엔드 또는 백엔드 파서(Papaparse 등)를 통해 텍스트 데이터 객체화.
3. 데이터 유효성 실시간 검증 실행.

### 2.2 유효성 검증 규칙 (Validation Rules)
- **필수값 누락 여부**: 이메일, 이름, 역할은 비어 있을 수 없음.
- **이메일 형식 검증**: 정규식을 이용해 올바른 이메일 주소인지 판별.
- **중복 검사**: 소프트 삭제되지 않은 활성 사용자의 이메일과 중복되는지 검증. 삭제된 계정과 같은 이메일은 신규 독립 계정으로 등록 가능.
- **역할 정의**: 역할 필드는 반드시 `EXPERT` 또는 `STARTUP`이어야 함.
- *오류 제어*: 오류 행이 발견될 경우, 해당 데이터를 DB에 넣지 않고 에러가 발생한 행(Line) 번호와 구체적인 사유를 팝업창으로 리스트업하여 표시. (예: `3번 라인: 이메일 형식이 올바르지 않습니다.`)

### 2.3 일괄 등록 후 인증 안내 규칙
- CSV 일괄 등록 시 사용자별 장기 Access Code를 생성하지 않습니다.
- 무료 운영 모드에서는 운영본부가 전체 참가자에게 동일한 로그인 URL과 `이름 + 휴대전화번호` 로그인 절차를 안내합니다.
- OTP 자동 발송은 기본 로그인 경로가 아니라 유료 알림 확장 모드 또는 진단용 후속 기능으로 분리합니다.
- 연락처 누락 또는 발송 실패 사용자는 관리자 화면에서 필터링하고 개별 수정할 수 있어야 합니다.
- 관리자는 보안 사고 또는 사용자 요청 시 `session_version`을 증가시켜 해당 참가자의 기존 세션을 모두 무효화할 수 있습니다.
- 현장 예외 처리용 1회용 로그인 링크는 본인 확인 후에만 발급하고 발급자·대상·만료 시각·사용 시각을 감사 로그에 남깁니다.

### 2.4 프로필 및 파일 관리
- 스타트업: 기업명, 대표자명, 담당자명, 연락처, 기업 소개, 홈페이지, 최대 3개 관심 분야, 소개서 PDF.
- 전문가: 이름, 소속, 직책, 연락처, 소개글, 최대 3개 전문 분야, 프로필 사진.
- 참가자는 소개글과 연락처 등 기초 정보만 수정할 수 있고 기업명과 이메일 등 식별값은 관리자만 수정합니다.
- 파일은 Supabase Storage에 저장하고 권한이 적용된 Signed URL로 제공합니다.

## 3. 코드 설계 가이드라인
- **컴포넌트 분할**:
  - 메인 관리 페이지: `ParticipantDbView.tsx` (스타트업 및 전문가 역할 고정 공통 화면)
  - CSV 업로더: `CsvBulkUploader` (CSV 검증 및 일괄 DB 삽입 모달)
  - 개별 추가 폼: `UserDetailModal.tsx` (개별 생성 및 편집 폼 모달)
  - 테이블 컬럼 정의: `participantColumns.tsx` (검색 키워드 추출 및 정렬 맵 정의)
  - 테이블 셀 렌더러: `participantCells.tsx` (성능 및 핫리로드 최적화를 위한 셀 분리 컴포넌트)
- **최대 라인 제한**:
  - CSV 데이터 파싱 및 행(line) 유효성 검증 로직은 `lib/csv.ts` 및 `lib/userCsv.ts`로, API 연동은 `hooks/useUsers.ts`로 관심사를 분리하여 파일당 **500줄 이하** 규정을 준수합니다.
- **스타일 제약**: 모바일 환경에서도 테이블 데이터의 가시성을 확보하기 위해 공통 `DataTable`에 부드러운 가로 스크롤(`overflow-x-auto`) 및 너비 정의(`minWidthClass="min-w-[960px]"`)를 필수로 지정하여 모바일 현장 기기 대응 레이아웃을 적용합니다.
