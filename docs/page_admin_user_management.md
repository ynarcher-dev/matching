# [기능 명세] 관리자 - 참가자 DB 관리 및 CSV 일괄 등록 (Admin User Management)

본 문서는 전문가 및 스타트업 참가자 데이터베이스를 조회, 개별 등록 및 CSV 파일을 통해 일괄 업로드 관리하는 화면의 상세 사양을 정의합니다.

---

## 1. 참가자 DB 관리 화면 (User DB Management View)

### 1.1 UI/UX 개요
- **진입 경로**: 관리자 대시보드 메뉴의 "참가자 DB 관리" 클릭 시 진입 (`/admin/users`).
- **주요 기능**: 등록된 모든 전문가와 스타트업의 정보를 테이블 형태로 조회하며, 신규 유저 등록 및 수정이 가능합니다.

### 1.2 화면 구성 (Layout Spec)
1. **역할 탭 필터**: `스타트업 목록` | `전문가 목록`
2. **조작 영역**:
   - `개별 추가` 버튼 (입력 폼 모달 오픈).
   - `CSV 일괄 업로드` 버튼 (파일 선택기 활성화).
3. **참가자 테이블 리스트**:
   - 공통 컬럼: 이름, 이메일, 연락처, 역할, 인증 가능 채널, 최근 OTP 발송 상태, 마지막 로그인 시각, 등록 일시.
   - 스타트업 전용 컬럼: 기업명, 대표자명, 담당자명, 관심 분야, 소개서.
   - 전문가 전용 컬럼: 소속, 직책, 전문 분야, 프로필 사진.
   - 조작 컬럼: `수정` | `삭제` | `세션 무효화` | `1회용 로그인 링크 발급`

## 2. CSV 일괄 업로드 파서 및 검증 기능 (CSV Bulk Upload)

### 2.1 파일 업로드 워크플로우
1. 관리자가 "CSV 일괄 업로드" 클릭 후 템플릿 양식에 맞춰 작성된 `.csv` 파일 선택.
2. 프론트엔드 또는 백엔드 파서(Papaparse 등)를 통해 텍스트 데이터 객체화.
3. 데이터 유효성 실시간 검증 실행.

### 2.2 유효성 검증 규칙 (Validation Rules)
- **필수값 누락 여부**: 이메일, 이름, 역할은 비어 있을 수 없음.
- **이메일 형식 검증**: 정규식을 이용해 올바른 이메일 주소인지 판별.
- **중복 검사**: 소프트 삭제되지 않은 활성 사용자의 이메일과 중복되는지 검증. 삭제된 계정과 같은 이메일은 신규 독립 계정으로 등록 가능.
- **역할 정의**: 역할 필드는 반드시 `EXPERT` 또는 `STARTUP`이어야 함.
- *오류 제어*: 오류 행이 발견될 경우, 해당 데이터를 DB에 넣지 않고 에러가 발생한 행(Line) 번호와 구체적인 사유를 팝업창으로 리스트업하여 표시. (예: `3번 라인: 이메일 형식이 올바르지 않습니다.`)

### 2.3 일괄 등록 후 인증 안내 규칙
- CSV 일괄 등록 시 사용자별 장기 Access Code를 생성하지 않습니다.
- 관리자는 등록 결과에서 이메일·휴대전화 등 OTP 수신 가능 채널과 누락·형식 오류를 확인합니다.
- 운영본부는 전체 참가자에게 동일한 로그인 URL과 `"등록된 연락처로 인증번호 받기"` 절차를 일괄 안내합니다.
- OTP는 사용자가 로그인할 때 요청하며, 시스템이 등록 연락처로 자동 발송합니다.
- 연락처 누락 또는 발송 실패 사용자는 관리자 화면에서 필터링하고 개별 수정할 수 있어야 합니다.
- 관리자는 보안 사고 또는 사용자 요청 시 `session_version`을 증가시켜 해당 참가자의 기존 세션을 모두 무효화할 수 있습니다.
- 현장 예외 처리용 1회용 로그인 링크는 본인 확인 후에만 발급하고 발급자·대상·만료 시각·사용 시각을 감사 로그에 남깁니다.

### 2.4 프로필 및 파일 관리
- 스타트업: 기업명, 대표자명, 담당자명, 연락처, 기업 소개, 홈페이지, 최대 3개 관심 분야, 소개서 PDF.
- 전문가: 이름, 소속, 직책, 연락처, 소개글, 최대 3개 전문 분야, 프로필 사진.
- 참가자는 소개글과 연락처 등 기초 정보만 수정할 수 있고 기업명과 이메일 등 식별값은 관리자만 수정합니다.
- 파일은 Supabase Storage에 저장하고 권한이 적용된 Signed URL로 제공합니다.

## 3. 코드 설계 가이드라인
- **컴포넌트 분할**:
  - 메인 관리 페이지: `UserManager.jsx` (역할 필터링 및 리스트 조회)
  - CSV 업로더 모달: `CsvBulkUploader.jsx` (드래그앤드롭 파일 업로드 및 에러 검증 인터페이스 담당)
  - 개별 추가 폼: `UserDetailModal.jsx` (CRUD 폼 컴포넌트)
- **최대 라인 제한**:
  - CSV 파싱 로직 및 DB Bulk Insert 로직은 프론트엔드 유틸리티 (`utils/csvParser.js`) 또는 Supabase Database Function으로 완전히 이전하여 단일 파일의 길이를 **500줄 이하**로 통제합니다.
- **스타일 제약**: 모바일 기기에서도 테이블 가로 스크롤(`overflow-x-auto`)을 부드럽게 구현하여 모바일 현장 스케줄 변경 시 표가 화면 밖으로 넘치거나 깨지지 않도록 레이아웃을 잡습니다.
