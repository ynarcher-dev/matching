# [기능 명세] 관리자 - 행사 상세 및 운영 대시보드 (Admin Event Detail & Dashboard)

본 문서는 특정 행사의 운영 대시보드 화면 상세 사양을 정의합니다. 참가자 배치 상태 조회, 수동 예약 강제 조정 및 실시간 진행 모니터링 기능이 포함됩니다.

---

## 1. 운영 대시보드 메인 화면 (Event Detail Dashboard)

### 1.1 UI/UX 개요
- **진입 경로**: 행사 카드에서 상세 보기 클릭 시 진입 (`/admin/events/:eventId`).
- **상태 전이 화면 대응**:
  - `DRAFT(대기)` 단계: 참가 스타트업 및 전문가 지정, 미팅 타임 슬롯 배치 기초 매핑 탭 노출.
  - `BOOKING(예약)` 단계: 실시간 스타트업 예약 진척도 및 미예약 기업 현황 통계 화면 노출.
  - `ALLOCATION(배치 조율)` 단계: 스타트업 직접 예약을 차단하고 AI 자동 배치 제안 검토와 관리자 수동 조정 기능 노출.
  - `PROGRESS(진행)` 단계: 실시간 스케줄 타임라인 활성화, 각 전문가별 테이블 출석 상태 감시 화면 노출.
  - `FINISHED(종료)` 단계: 결과 데이터 조회, 만족도 점수 리포트 그래프, 엑셀 다운로드 버튼 노출.
  - `CANCELLED(취소)` 단계: 변경 기능을 잠그고 취소 행사 정보와 기존 이력만 조회.

### 1.2 화면 탭 구조 및 권한 매핑 (Tab Structure & Capabilities)

행사 상세 화면의 운영 탭은 다음 10개 탭으로 구성되며, 사용자의 권한 수준(`TAB_CAPABILITY`)에 따라 탭 노출 여부가 제한됩니다. RLS 및 RPC 단에서 최종 게이팅이 작동하며, 권한 등급은 최고관리자(`is_super_admin`) 또는 행사별 배정 권한(`OWNER`, `MANAGER`, `STAFF`, `VIEWER`)에 기초합니다.

1. **참가 스타트업** (`startups`, 권한: `manage` = OWNER / MANAGER)
   - 행사에 참가할 스타트업 후보를 지정, 일괄 추가 및 제외 처리하는 패널.
2. **참가 전문가** (`experts`, 권한: `manage` = OWNER / MANAGER)
   - 참가 전문가를 배정하고, 전문가별 행사 기본 테이블(`default_table_id`)을 지정.
3. **테이블 세팅** (`tables`, 권한: `manage` = OWNER / MANAGER)
   - 행사장 내 상담 테이블(`event_tables`)의 코드와 위치를 추가, 수정, 삭제 관리.
4. **예약·강제조정** (`booking`, 권한: `view` = OWNER / MANAGER / STAFF / VIEWER)
   - 스타트업 예약율 통계와 미예약 기업 명단을 제공하고, AI 자동배치 화면 진입 및 슬롯 강제 배정/취소 제어. (수동 배정 및 조정 영역은 `manage` 권한으로 내부 게이팅)
5. **진행 현황** (`progress`, 권한: `staff` = OWNER / MANAGER / STAFF)
   - 실시간 스케줄 타임그리드 시트(`TimeGridSheet`)를 제공하며, 슬롯별 출석 처리 및 상담 진행 상황 모니터링.
6. **상담일지** (`counseling`, 권한: `view` = OWNER / MANAGER / STAFF / VIEWER)
   - 작성된 상담일지 결과 목록 및 요약을 제공하며, 개별 평가 텍스트 코멘트 상세 조회 가능. (상담일지 설정 모달은 `manage` 권한으로 게이팅)
7. **행사 만족도** (`survey`, 권한: `view` = OWNER / MANAGER / STAFF / VIEWER)
   - 스타트업 대상 동적 설문 만족도 리포트 그래프 조회 및 CSV 내보내기 지원. (설문 문항 편집은 `manage` 권한으로 게이팅)
8. **전문가 만족도** (`expert-survey`, 권한: `view` = OWNER / MANAGER / STAFF / VIEWER)
   - 전문가 대상 동적 설문 만족도 리포트 그래프 조회 및 CSV 내보내기 지원.
9. **증빙사진** (`photos`, 권한: `staff` = OWNER / MANAGER / STAFF)
   - 현장 스태프 및 기업별로 업로드된 현장 사진첩 데이터 조회 및 관리.
10. **행사알림** (`notifications`, 권한: `view` = OWNER / MANAGER / STAFF / VIEWER)
    - 행사 관련 카카오톡/SMS 발송 로그 목록 및 통계 확인. (행사별 발송 정책 설정 모달은 `manage` 권한으로 게이팅)

운영형 탭(상담일지, 만족도, 알림 등)은 결과 및 리포트를 먼저 보여주는 형태로 구성하고, 설정은 별도 탭으로 확장하지 않고 우측 상단의 `설정` 버튼을 클릭해 모달 창을 통해 관리합니다.

## 2. 참가자 매핑 및 현황 모니터링 (대기 및 예약 단계)

### 2.1 참가자 지정 기능 (DRAFT)
- **전문가 지정 탭**:
  - 해당 행사에 참여할 전문가 리스트 체크박스로 선택 및 추가.
- **스타트업 지정 탭**:
  - 해당 행사에 참여할 스타트업 리스트 체크박스로 선택 및 추가.
  - 각 스타트업의 자문 대상 분야 매핑 설정.
- **테이블 지정**:
  - `event_tables`에서 행사장 테이블과 위치 설명을 관리합니다.
  - 전문가별 기본 테이블을 지정하고 필요하면 개별 슬롯에서 오버라이드합니다.

### 2.2 예약 현황 통계 대시보드 (BOOKING)
- **통계 위젯**:
  - 전체 타임 슬롯 대비 예약율 (원형 차트 또는 프로그레스 바).
  - 미예약 스타트업 수치 / 예약 완료 스타트업 수치.
- **미예약자 긴급 알림**:
  - 예약 횟수가 0회인 스타트업에 대해 수동 알림톡 재발송 버튼 제공.
  - 자동 마감 임박 시 AI 자동배치 화면(`admin/events/:eventId/ai-allocation`)으로 이동할 수 있는 단추 활성화.

## 3. 실시간 매칭 타임라인 및 예약 강제 조정 (진행 단계)

### 3.1 실시간 진행 타임라인
- **그리드 시간표 (Time-Grid Sheet)**:
  - Y축: 전문가 이름 (테이블 번호, 소속 정보 표시)
  - X축: 시간대 슬롯 (예: 10:00, 10:40, 11:20 ...)
  - 각 슬롯 카드 내용: 스타트업명, 예약 유형 배지(수동 예약, AI 자동배치, 관리자 강제 배정), 상담 진행 상태 배지 (`WAITING`, `IN_PROGRESS`, `COMPLETED`).
- **출석(Attendance) 및 노쇼 체크 기능**:
  - 전문가와 스타트업의 출석 여부를 슬롯별로 각각 개별 표시 및 조정합니다.
  - **3단 세그먼트 컨트롤** `[미정(–) | 출석(✓) | 불참(✕)]`을 제공하여, 관리자나 스태프가 현장 상황에 맞춰 원클릭으로 대리 출석 처리할 수 있습니다. 잘못 클릭한 경우에는 다시 미정(–)을 클릭하여 초기화(기록 삭제)가 가능합니다.
  - 스타트업이 불참한 경우 노쇼 처리(`mark_no_show` RPC)를 수행하며, 노쇼 처리 시 구체적인 사유를 필수로 입력하도록 유도합니다.
  - 진행 단계(`PROGRESS`) 행사에서는 7초 주기(`refetchInterval`)의 폴링 방식으로 화면 데이터를 근실시간 자동 갱신합니다.
  - *비고*: QR 스캔 기반의 출석 체크 기능은 현장 운영의 단순화를 위해 도입 보류 및 제외되었습니다.

### 3.2 관리자 제어 및 강제 조정 (Force Adjustment)
- **스타트업 자율 예약 제어 토글**:
  - 관리자 대시보드 상단에 `"스타트업 자율 예약 허용"` 토글 스위치가 제공됩니다.
  - 이 토글이 켜져 있으면 `ALLOCATION` 또는 `PROGRESS` 단계에서도 스타트업이 자율적으로 본인 예약을 취소 및 변경할 수 있으며, 꺼져 있으면 직접 조작이 완전 차단됩니다.
- **동작 방식**:
  - 빈 슬롯 카드를 더블 클릭하거나 "강제 배정" 버튼 클릭 시 팝업창 활성화.
  - 배정할 스타트업 리스트 셀렉트 노출 (이때 이미 해당 시간대에 다른 예약을 소지한 스타트업은 회색으로 비활성화하거나 경고 메시지 노출).
- **중복 방지 유효성 검증**:
  - SQL 트랜잭션 및 API 단에서 `matching_slots` 내 동일 `startup_id` 또는 동일 `expert_id`가 지정한 `start_time`과 `end_time` 사이에 이미 존재하는지 판단하여 예외 차단.
  - 경고 문구: `"해당 스타트업은 이미 동일 시간대 다른 전문가와 매칭이 예약되어 있습니다."`
  - 관리자는 최대 상담 횟수만 예외적으로 우회할 수 있으며, 동시간 중복과 실제 적용 테이블 충돌은 우회할 수 없습니다.
  - 강제 변경과 취소는 DB RPC로 처리하고 `booking_history` 및 `audit_logs`에 기록합니다.

### 3.3 결과 리포트 및 만족도 (FINISHED)
- 스타트업과 전문가는 행사 전체 및 상담 관련 동적 만족도 문항에 대해 행사당 1회 제출할 수 있으며, 제출 후 수정할 수 없습니다.
- 만족도 조사 및 결과 수집은 동적 설문 빌더(`SurveyBuilderPanel`)를 통해 행사 상세 "만족도 설정" 탭에서 스타트업과 전문가 역할을 각각 분리하여 자유롭게 질문(5개 유형: 객관식 5척도, 객관식 다중선택, 객관식 단일선택, 주관식 단락, 주관식 한줄)을 구성해 관리합니다.
- 상담일지 및 스코어카드 평가 점수는 관리자 전용입니다. 스타트업에는 전문가가 상담 일지 작성 시 공개를 허용한(`is_public = true`) 자유 코멘트 의견만 행사 종료 후 노출됩니다.
- **종합 엑셀 다운로드 기능**:
  - `exceljs` 라이브러리를 기반으로 생성된 통합 워크북을 제공합니다.
  - 제공되는 5개의 독립된 시트:
    1. **예약 현황**: 배정 슬롯 시간, 매칭 전문가, 스타트업 및 진행상태 정보.
    2. **출석 현황**: 슬롯별 전문가 및 스타트업의 실제 출석 여부 기록.
    3. **상담 결과**: 전문가가 작성한 스코어카드 다차원 점수(기술성, 전문성 등) 및 정성적 의견.
    4. **만족도 결과**: 참가자들로부터 수집된 동적 설문 답변 일괄 정리.
    5. **참가자 명단**: 행사에 소속된 기업/전문가별 기본 인적사항 명단.
  - 엑셀 다운로드는 일반 관리자 이상의 보기 권한을 소지한 계정에서 자유롭게 수행할 수 있습니다.

### 3.4 공통 데이터 테이블 기준

- 행사 상세 안의 목록형 데이터는 기본적으로 테이블을 우선한다.
- 페이지네이션 기본값은 한 페이지 30개다.
- 검색, 필터, 정렬은 각 테이블의 운영 목적에 맞게 제공한다.
- 모바일 현장 대응 화면은 별도 카드/리스트형을 허용하되, 백오피스 대량 관리 화면은 테이블형을 기본으로 한다.

## 4. 코드 설계 가이드라인
- **컴포넌트 분할**:
  - 대시보드 페이지: `EventDetailDashboard.jsx` (행사 상태에 따라 분기 렌더링)
  - 시간표 렌더러: `TimeGridSheet.jsx` (Tailwind Grid 기반 테이블 형태로 스케줄 렌더링)
  - 스케줄 조정 모달: `ForceBookingModal.jsx` (예약 중복 체크 로직 포함)
- **최대 라인 제한**:
  - 시간표 스케줄 데이터 파싱 로직 및 DB 리얼타임 구독 모듈을 Custom Hook (`useEventDashboard.js`)으로 분리하여 파일당 **500줄 규정**을 충족합니다.
