# [기능 명세] 관리자 - 행사 상세 및 운영 대시보드 (Admin Event Detail & Dashboard)

본 문서는 특정 행사의 운영 대시보드 화면 상세 사양을 정의합니다. 참가자 배치 상태 조회, 수동 예약 강제 조정 및 실시간 진행 모니터링 기능이 포함됩니다.

---

## 1. 운영 대시보드 메인 화면 (Event Detail Dashboard)

### 1.1 UI/UX 개요
- **진입 경로**: 행사 카드에서 상세 보기 클릭 시 진입 (`/admin/events/:eventId`).
- **상태 전이 화면 대응**:
  - `DRAFT(대기)` 단계: 참가 스타트업 및 전문가 지정, 미팅 타임 슬롯 배치 기초 매핑 탭 노출.
  - `BOOKING(예약)` 단계: 실시간 스타트업 예약 진척도 및 미예약 기업 현황 통계 화면 노출.
  - `ALLOCATION(배치 조율)` 단계: 스타트업 직접 예약을 차단하고 AI 자동 배치 제안 검토와 관리자 수동 조정 기능 노출.
  - `PROGRESS(진행)` 단계: 실시간 스케줄 타임라인 활성화, 각 전문가별 테이블 출석 상태 감시 화면 노출.
  - `FINISHED(종료)` 단계: 결과 데이터 조회, 만족도 점수 리포트 그래프, 엑셀 다운로드 버튼 노출.
  - `CANCELLED(취소)` 단계: 변경 기능을 잠그고 취소 행사 정보와 기존 이력만 조회.

---

## 2. 참가자 매핑 및 현황 모니터링 (대기 및 예약 단계)

### 2.1 참가자 지정 기능 (DRAFT)
- **전문가 지정 탭**:
  - 해당 행사에 참여할 전문가 리스트 체크박스로 선택 및 추가.
- **스타트업 지정 탭**:
  - 해당 행사에 참여할 스타트업 리스트 체크박스로 선택 및 추가.
  - 각 스타트업의 자문 대상 분야 매핑 설정.
- **테이블 지정**:
  - `event_tables`에서 행사장 테이블과 위치 설명을 관리합니다.
  - 전문가별 기본 테이블을 지정하고 필요하면 개별 슬롯에서 오버라이드합니다.

### 2.2 예약 현황 통계 대시보드 (BOOKING)
- **통계 위젯**:
  - 전체 타임 슬롯 대비 예약율 (원형 차트 또는 프로그레스 바).
  - 미예약 스타트업 수치 / 예약 완료 스타트업 수치.
- **미예약자 긴급 알림**:
  - 예약 횟수가 0회인 스타트업에 대해 수동 알림톡 재발송 버튼 제공.
  - 자동 마감 임박 시 AI 자동배치 화면(`admin/events/:eventId/ai-allocation`)으로 이동할 수 있는 단추 활성화.

## 3. 실시간 매칭 타임라인 및 예약 강제 조정 (진행 단계)

### 3.1 실시간 진행 타임라인
- **그리드 시간표 (Time-Grid Sheet)**:
  - Y축: 전문가 이름 / 테이블 번호
  - X축: 시간대 슬롯 (예: 10:00, 10:40, 11:20 ...)
  - 각 슬롯 카드 내용: 스타트업명, 예약 유형(수동: 민트색, AI: 보라색, 강제: 주황색), 상담 진행 상태 배지 (대기중, 진행중, 완료).
- **출석(Attendance) 체크 기능**:
  - 전문가와 스타트업의 출석을 슬롯별로 각각 표시합니다.
  - 전문가는 대시보드 버튼 터치 또는 본인 QR로 체크인할 수 있으며, 스타트업 출석은 관리자/스태프가 대시보드 화면에서 원클릭으로 일괄 체크하거나 전문가가 본인의 대시보드 카드에서 직접 확인할 수 있습니다.
  - 보조 수단으로 QR 스캔 출석 체크 또한 지원합니다.

### 3.2 관리자 제어 및 강제 조정 (Force Adjustment)
- **스타트업 자율 예약 제어 토글**:
  - 관리자 대시보드 상단에 `"스타트업 자율 예약 허용"` 토글 스위치가 제공됩니다.
  - 이 토글이 켜져 있으면 `ALLOCATION` 또는 `PROGRESS` 단계에서도 스타트업이 자율적으로 본인 예약을 취소 및 변경할 수 있으며, 꺼져 있으면 직접 조작이 완전 차단됩니다.
- **동작 방식**:
  - 빈 슬롯 카드를 더블 클릭하거나 "강제 배정" 버튼 클릭 시 팝업창 활성화.
  - 배정할 스타트업 리스트 셀렉트 노출 (이때 이미 해당 시간대에 다른 예약을 소지한 스타트업은 회색으로 비활성화하거나 경고 메시지 노출).
- **중복 방지 유효성 검증**:
  - SQL 트랜잭션 및 API 단에서 `matching_slots` 내 동일 `startup_id` 또는 동일 `expert_id`가 지정한 `start_time`과 `end_time` 사이에 이미 존재하는지 판단하여 예외 차단.
  - 경고 문구: `"해당 스타트업은 이미 동일 시간대 다른 전문가와 매칭이 예약되어 있습니다."`
  - 관리자는 최대 상담 횟수만 예외적으로 우회할 수 있으며, 동시간 중복과 실제 적용 테이블 충돌은 우회할 수 없습니다.
  - 강제 변경과 취소는 DB RPC로 처리하고 `booking_history` 및 `audit_logs`에 기록합니다.

### 3.3 결과 리포트 및 만족도 (FINISHED)
- 스타트업과 전문가는 행사 전체에 대한 고정 4문항 5점 평가와 자유 의견을 행사당 1회 제출할 수 있으며 제출 후 수정할 수 없습니다.
- 상담 평가 점수는 관리자 전용입니다. 스타트업에는 `is_public = true`인 텍스트 코멘트만 행사 종료 후 공개합니다.
- 엑셀은 상담 결과, 만족도, 예약 변경·취소 이력을 독립 시트로 제공합니다.
- 엑셀 다운로드는 `ADMIN`(일반 관리자) 이상 권한을 가진 계정에서 모두 사용이 가능합니다. (MFA 적용 권장)

## 4. 코드 설계 가이드라인
- **컴포넌트 분할**:
  - 대시보드 페이지: `EventDetailDashboard.jsx` (행사 상태에 따라 분기 렌더링)
  - 시간표 렌더러: `TimeGridSheet.jsx` (Tailwind Grid 기반 테이블 형태로 스케줄 렌더링)
  - 스케줄 조정 모달: `ForceBookingModal.jsx` (예약 중복 체크 로직 포함)
- **최대 라인 제한**:
  - 시간표 스케줄 데이터 파싱 로직 및 DB 리얼타임 구독 모듈을 Custom Hook (`useEventDashboard.js`)으로 분리하여 파일당 **500줄 규정**을 충족합니다.
