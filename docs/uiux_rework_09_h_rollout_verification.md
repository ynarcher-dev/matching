# UI/UX Rework 09-H - Rollout and Verification

## 1. 목표

9번째 UI/UX 개편을 안전하게 적용하기 위한 순서와 검증 기준을 정의한다.

## 2. 적용 순서

1. 디자인 토큰 확장
2. Button, Badge, Tabs, Select 계층 추가
3. Sidebar 접기 및 Header 정리
4. 관리자 목록 화면부터 공통 컴포넌트 적용
5. 행사 상세 탭과 설정 모달 정리
6. 스타트업 예약 모바일 UX 개선
7. 전문가/현장 모바일 UX 개선
8. 전체 반응형 검증과 문서 업데이트

## 3. 회귀 검증 기준

기능:

- 로그인 역할 전환과 로그인 성공
- 행사 목록 조회, 검색, 필터, 생성/수정/취소
- 행사 상세 탭 이동과 권한별 표시
- 참가자 추가/제외, 기본 테이블 변경
- 예약/변경/취소/강제 배정
- 출석/불참/상담 시작/상담일지 저장
- 만족도 제출과 결과 조회
- 알림 설정 저장과 로그 재시도
- 사진 촬영/선택/업로드/삭제

반응형:

- 360px 모바일
- 768px 태블릿
- 1024px 노트북
- 1440px 데스크톱

상태:

- loading
- empty
- error
- disabled
- success
- permission denied

## 4. 명령 검증

기본:

```bash
npm run lint
npm run typecheck
npm run build
npm run test
```

화면 검증:

- 로컬 dev server 실행
- 브라우저에서 주요 화면 모바일/데스크톱 확인
- 텍스트 겹침, 버튼 overflow, 테이블 가로 스크롤, 모달 높이 확인

## 5. 완료 기준

- 기능 회귀 없이 공통 컴포넌트가 적용된다.
- 모든 주요 화면에서 모바일/데스크톱 기본 사용 흐름이 가능하다.
- 새 UI 규칙이 문서와 코드에서 같은 이름으로 사용된다.
- 구현 완료 후 `development_status.md`의 9번째 작업 체크리스트가 업데이트된다.
