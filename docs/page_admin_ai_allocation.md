# [기능 명세] 관리자 - AI 자동배치 제안 및 검토 화면 (Admin AI Allocation)

본 문서는 예약을 완료하지 못한 잔여 스타트업들을 빈 타임 슬롯에 알고리즘으로 자동 매칭하고, 최종 확정하기 전에 관리자가 검토하는 AI 자동배치 화면의 상세 사양을 정의합니다.

---

## 1. AI 자동배치 실행 및 제안안 생성 프로세스

### 1.1 진입 경로 및 활성화 시점
- **진입 경로**: 행사 상세 페이지 예약 탭의 "AI 자동배치 생성" 버튼 클릭 시 진입 (`/admin/events/:eventId/ai-allocation`).
- **활성화 조건**: 행사 상태가 `ALLOCATION(배치 조율)` 단계일 때 사용 가능.

### 1.2 자동배치 동작 흐름 (Sequence)
1. **미예약 대상 추출**: 설정된 `max_sessions_per_startup`보다 적게 예약한 스타트업을 조회하고, 그중 0회 배정 기업을 우선합니다.
2. **잔여 타임 슬롯 추출**: 행사 내 전문가별로 스타트업이 매칭되지 않은(NULL) 예약 시간 슬롯 파악.
3. **분야 및 일정 적합도 스코어 연산**:
   - `S_field`: `event_participant_fields`를 우선 사용하고 미설정 시 `user_fields`를 사용해 분야 일치 여부를 계산.
   - `S_time`: 해당 스타트업이 매치하려는 시간대에 다른 행사의 세션을 이미 가지고 있는지 여부 검사.
4. **그리디(Greedy) 임시 배정**:
   - 랜덤 요소 없이 점수가 높은 순서로 임시 제안을 생성하여 동일 입력에 동일 결과를 보장.
   - 가용 슬롯과 제약이 허용하는 범위에서 0회 배정 기업을 우선하며, 분야 불일치 차선 배정은 경고 대상으로 표시.
5. **제안 저장**:
   - 확정 전 결과는 `matching_proposals`에 저장하고 실제 `matching_slots`는 변경하지 않음.
   - 미배치 기업에는 슬롯 부족, 시간 충돌, 적합 분야 없음 등의 사유를 저장.

## 2. AI 자동배치 제안안 검토 화면 (Allocation Preview UI)

### 2.1 화면 구성 (UX Spec)
- **상단 조작 바**:
  - 버튼: `배치 제안안 확정 (Apply)` | `새로고침 (Re-calculate)` | `돌아가기 (Cancel)`
  - 요약 문구: `"미매칭 스타트업 N개사 대상 M개의 자동 매칭 슬롯이 제안되었습니다."`
- **임시 배정 시간표 (Preview Time-Grid)**:
  - 캘린더/스케줄러 테이블 뷰.
  - **컬러 코드 시각화**:
    - 기존 수동 예약 확정 건: 민트색 배경
    - **AI 제안안 배정 건**: **연보라색(Light Purple) 배경 + "AI 제안" 라벨** 표기.
    - 충돌 발생 우려 슬롯 (수동 체크 필요 시): 붉은색 보더 라인.
    - 분야 불일치 차선 배정: 경고 아이콘과 `"분야 불일치"` 텍스트 라벨 표기.
- **수동 드래그 앤 드롭 또는 변경 기능**:
  - 관리자가 임시 배정된 연보라색 카드를 잡아서 다른 빈 슬롯으로 드롭하여 매칭 관계 변경 가능.
  - 마우스 오버 시 스타트업명, 전문가명, 추천 매칭 점수 점수(예: `적합도 95%`) 툴팁 노출.
  - 관리자가 수동 변경한 제안은 `is_locked = true`로 저장하여 재계산 시 보존.
  - 배정되지 못한 스타트업과 미배치 사유를 별도 목록으로 제공.

### 2.2 최종 확정 처리 (Apply Process)
- 관리자가 "배치 제안안 확정" 버튼을 클릭하면:
  1. 전체 제안안을 단일 DB RPC 트랜잭션으로 검증하고 `matching_slots`에 반영.
  2. `booking_type` 컬럼에 `'AUTO_AI'` 문자열 저장.
  3. 성공 완료 메시지와 함께 행사 상세 페이지로 리다이렉트.
  4. 대상 스타트업 및 전문가에게 알림톡(카카오톡) 배치 알림 자동 발송 트리거 작동.
  5. 충돌이 발생한 제안 슬롯은 제외하고 정상 슬롯만 부분 확정하여 반영할 수 있으며, 충돌이 발생한 구체적인 사유와 대상은 요약 리포트로 화면에 명확히 인지시킵니다.

## 3. 코드 설계 가이드라인
- **컴포넌트 분할**:
  - 메인 매칭 뷰: `AiAllocationReview.jsx`
  - 드래그앤드롭 스케줄 카드: `DraggableSlotCard.jsx`
  - 매칭 제안 툴팁: `AllocationTooltip.jsx`
- **최대 라인 제한**:
  - 드래그 앤 드롭 UI는 프론트엔드 컴포넌트로 분리하되, 최종 점수 계산과 확정 검증은 서버 함수 또는 DB RPC에서 수행합니다.
