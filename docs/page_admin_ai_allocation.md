# [기능 명세] 관리자 - AI 자동배치 제안 및 검토 화면 (Admin AI Allocation)

본 문서는 예약을 완료하지 못한 잔여 스타트업들을 빈 타임 슬롯에 알고리즘으로 자동 매칭하고, 최종 확정하기 전에 관리자가 검토하는 AI 자동배치 화면의 상세 사양을 정의합니다.

---

## 1. AI 자동배치 실행 및 제안안 생성 프로세스

### 1.1 진입 경로 및 활성화 시점
- **진입 경로**: 행사 상세 페이지 예약 탭의 "AI 자동배치 생성" 버튼 클릭 시 진입 (`/admin/events/:eventId/ai-allocation`).
- **활성화 조건**: 행사 상태가 `ALLOCATION(배치 조율)` 단계일 때 사용 가능.

### 1.2 자동배치 동작 흐름 (Sequence)
1. **미예약 대상 추출**: 설정된 `max_sessions_per_startup`보다 적게 예약한 스타트업을 조회하고, 그중 0회 배정 기업을 우선합니다.
2. **잔여 타임 슬롯 추출**: 행사 내 전문가별로 스타트업이 매칭되지 않은(NULL) 예약 시간 슬롯 파악.
3. **분야 및 일정 적합도 스코어 연산**:
   - `S_field`: `event_participant_fields`를 우선 사용하고 미설정 시 `user_fields`를 사용해 분야 일치 여부를 계산.
   - `S_time`: 해당 스타트업이 매치하려는 시간대에 다른 행사의 세션을 이미 가지고 있는지 여부 검사.
4. **그리디(Greedy) 임시 배정**:
   - 랜덤 요소 없이 점수가 높은 순서로 임시 제안을 생성하여 동일 입력에 동일 결과를 보장.
   - 가용 슬롯과 제약이 허용하는 범위에서 0회 배정 기업을 우선하며, 분야 불일치 차선 배정은 경고 대상으로 표시.
5. **제안 저장**:
   - 확정 전 결과는 `matching_proposals`에 저장하고 실제 `matching_slots`는 변경하지 않음.
   - 미배치 기업에는 슬롯 부족, 시간 충돌, 적합 분야 없음 등의 사유를 저장.

## 2. AI 자동배치 제안안 검토 화면 (Allocation Preview UI)

### 2.1 화면 구성 (UX Spec)
- **상단 조작 바 (`AllocationToolbar`)**:
  - 버튼: `배치 제안안 확정` | `배치 제안 재계산` | `돌아가기` (행사 상세로 이동)
  - 요약 문구: AI 자동배치 결과(매칭 M건, 미매칭 N건, 고정 보존 K건, 빈 슬롯 S개) 요약 출력.
- **임시 배정 시간표 (`AllocationSlotBoard`)**:
  - 캘린더/스케줄러 테이블 뷰.
  - **컬러 코드 시각화**:
    - 기존 수동 예약 확정 건: 민트색 배경
    - **AI 제안안 배정 건**: **연보라색(Light Purple) 배경 + "AI 제안" 라벨** 표기.
    - 충돌 발생 우려 슬롯 (수동 조정 필요 시): 붉은색 보더 라인.
    - 분야 불일치 차선 배정: 경고 아이콘과 `"분야 불일치"` 텍스트 라벨 표기.
- **수동 고정 및 시간 변경 조정 기능**:
  - 드래그 앤 드롭 대신, 각 슬롯 카드의 드롭다운 선택 또는 락 클릭을 통해 제안 상태를 직접 제어합니다.
  - **슬롯 제안 락 토글**: 자물쇠 아이콘 클릭 시 `is_locked` 상태가 토글(`onToggleLock`)됩니다. 고정(Lock)된 제안은 "재계산" 실행 시에도 변경되지 않고 그대로 유지됩니다.
  - **슬롯 위치 이동**: 슬롯 이동 UI(`onMove`)를 제공하여, 관리자가 특정 AI 제안 건을 다른 빈 슬롯으로 수동 이동 조정할 수 있습니다.
  - 마우스 오버 또는 클릭 시 스타트업명, 전문가명, 분야 적합도 등의 정보 툴팁 노출.
- **미배치 패널 (`UnmatchedPanel`)**:
  - 조건상 예약되지 못한 스타트업 목록과 각각의 미배치 사유(예: "시간 충돌", "테이블 충돌", "한도 초과" 등)를 별도 목록으로 하단에 일괄 제공합니다.

### 2.2 최종 확정 처리 (Apply Process)
- 관리자가 "배치 제안안 확정" 버튼을 클릭하면 `ConfirmModal` 창이 노출되며 확인 후 확정이 진행됩니다.
  1. 전체 제안안에 대해 개별 서브트랜잭션으로 유효성을 재검증하고 `confirm_ai_proposals` RPC를 실행하여 `matching_slots`에 반영합니다.
  2. 확정된 슬롯의 `booking_type` 컬럼에는 `'AUTO_AI'` 문자열이 저장됩니다.
  3. **부분 확정 및 충돌 제외 처리**: 검증 과정에서 시간대나 테이블의 실시간 충돌이 감지된 제안은 안전하게 제외(Skip)하고, 정상 건들만 부분 확정하여 반영합니다.
  4. **결과 리포트 출력**: 확정이 완료되면 즉시 화면에 확정 결과 리포트(`ConfirmReport`)를 가시화합니다. (성공 반영 건수, 제외 건수 및 제외 사유 리스트 명시)
  5. 대상 스타트업 및 전문가에게 알림 정책이 켜져 있을 경우 배치 완료 알림톡(카카오톡/SMS) 발송 트리거가 백그라운드에서 실행됩니다.

## 3. 코드 설계 가이드라인
- **컴포넌트 분할**:
  - 메인 매칭 뷰: `AiAllocationView.tsx`
  - 제안 툴바: `AllocationToolbar.tsx`
  - 스케줄 보드: `AllocationSlotBoard.tsx` (슬롯 고정 및 이동 기능 내장)
  - 슬롯 카드: `ProposalSlotCard.tsx`
  - 미배치 리스트: `UnmatchedPanel.tsx`
- **최대 라인 제한**:
  - 복잡한 AI 배치 계산 알고리즘 및 락 토글/이동 핸들러는 React Query 훅(`hooks/useAiAllocation.ts`)으로 모듈화하여 파일당 **500줄 규정**을 이행합니다.
