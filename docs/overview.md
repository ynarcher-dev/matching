## 2. 서비스 핵심 정의 및 목표

### 2.1 기획 배경
- 오프라인 비즈니스 매칭 행사 시 매칭 매니저들의 수기 엑셀 조율 부담을 덜고, 현장 불참(노쇼)과 스케줄 변동에 즉각 반응할 수 있는 모바일 웹 대시보드를 제공하여 행사 운영의 신속성을 극대화합니다.

### 2.2 사용자 권한 분류
1. **관리자 (Admin)**: 행사 개설, 참가자 등록(CSV), 스케줄 수동/AI 강제 배치, 현장 모니터링 및 만족도 종합 엑셀 데이터 추출.
2. **현장 스태프 (Staff)**: QR 또는 수동 검색을 통한 스타트업 출석 처리.
3. **전문가 (Expert)**: 배정된 시간표 조회, 본인 출석 및 상담 시작 처리, 모바일 스코어카드 작성과 최종 완료 처리.
4. **스타트업 (Startup)**: 등록된 이메일 또는 휴대전화로 일회용 인증번호(OTP) 로그인, 배정 카테고리 내 선착순 예약, 취소, 변경 조작.

## 3. 행사 상태 흐름 (Lifecycle Status)
행사의 상태(`status`)는 시간 스케줄에 맞춰 순차 변경되며 이에 따라 유저 화면이 다이내믹하게 연동됩니다.

1. **대기 (Draft)**: 행사 설정 구성 및 참가자(기업, 전문가) 매핑을 세팅하는 단계.
2. **예약 (Booking)**: 예약 가능 기한이 활성화되어 스타트업이 시간표를 직접 선택하는 단계.
3. **배치 조율 (Allocation)**: 스타트업 직접 예약을 닫고 관리자가 자동 배치 제안과 수동 조정을 확정하는 단계.
4. **진행 (Progress)**: 본 미팅이 시작되어 출석, 상담 시작, 타이머 및 상담일지 기능이 활성화되는 단계.
5. **종료 (Finished)**: 만족도 조사, 결과 리포트 및 원데이터 엑셀 다운로드 단계.
6. **취소 (Cancelled)**: 취소된 행사를 보존하되 일반 운영 대상에서 격리하는 상태.

## 4. 아키텍처 및 품질 기준 요약
- **기술 스택**: React (Vite SPA) + Supabase + Tailwind CSS (v4)
- **개발 한계 규정**: 모든 소스코드 파일은 **최대 500줄 이하**로 제어합니다.
- **스타일 규정**: 인라인 스타일 지정을 전면 금지하며, Tailwind CSS 클래스만을 이용해 모바일 반응형 페이지를 구축합니다.
- **연동 규칙**: 상세 DB 물리 설계는 [db_schema.md](./db_schema.md)를 상시 준수합니다.
- **서버 변경 규칙**: 인증, 예약, 출석, 상태 전환은 [security_transactions.md](./security_transactions.md)의 RLS 및 RPC 원칙을 준수합니다.

## 5. UI/UX 및 디자인 시스템 규격 (UI/UX Guidelines)
본 시스템은 현장의 가독성과 일관된 브랜드 아이덴티티를 유지하기 위해 아래 디자인 룰을 엄격히 준수합니다.

### 5.1 네비게이션 배치 (Left Side Menu)
- **데스크톱 환경**: 화면 **좌측에 고정된 메뉴 영역 (Left Navigation Menu, 너비 240px)**을 상시 배치하여 직관적인 페이지 이동을 보장합니다.
- **모바일/태블릿 환경**: 스크린 공간 확보를 위해 기본적으로 숨겨져 있으며, 좌측 상단 햄버거 메뉴 터치 시 **왼쪽에서 오른쪽으로 슬라이드인(Slide-in from Left)**하는 오버레이 드로어(Drawer) 형태로 제공합니다.

### 5.2 색상 배색 (Color Palette)
- **메인 브랜드 컬러 (Primary Accent)**: `#E22213` (선명한 레드)
  - 주요 쓰임새: 활성화된 탭 하이라이트, 중요 조작 버튼(예약 신청, 제출), 상태 표시 배지, 브랜드 로고 포인트.
- **서브 네츄럴 컬러 (Neutral Base)**: `#515151` (다크 그레이)
  - 주요 쓰임새: 좌측 메뉴 배경색, 메인 텍스트 컬러, 비활성화된 아이콘. 모든 카드와 입력창의 기본 테두리로 반복 사용하지 않습니다.
- **경계선 컬러 (Border Subtle)**: `#D4D4D4` 또는 동등한 중립 회색
  - 주요 쓰임새: 카드, 입력창, 표의 기본 구분선. 포커스·오류·선택 상태에서는 브랜드 컬러와 상태 컬러로 교체합니다.

### 5.3 자연스러운 시각 위계와 가시성
- 모든 요소를 굵은 선으로 둘러싸지 않고 여백, 배경색, 얇은 경계선, 제한적인 그림자를 조합해 정보 위계를 만듭니다.
- **명확한 대비 (Contrast)**: 배경색과 글자색의 대비를 확실히 주어 태양광 아래 야외나 밝은 실내에서도 모바일 텍스트가 또렷하게 읽히도록 구성합니다.
- **경계 강도 분리**: 일반 카드·입력창은 1px 중립 경계선을 사용하고, 선택·포커스·오류 등 상호작용 상태에만 강한 색을 사용합니다.
- **카드 표현**: 흰색 표면, 12~16px 모서리, 1px 중립 경계선을 기본으로 하며 필요한 경우에만 매우 약한 그림자를 허용합니다.
- **컴포넌트 우선순위**: Primary 버튼은 브랜드 컬러 면으로, 탭은 배경 기반 세그먼트 컨트롤로, 오류는 연한 상태 배경과 아이콘 또는 왼쪽 강조선으로 표현합니다.
- **직관적인 상태 분기**: 매칭 상태별(수동/AI/강제) 고대비 컬러와 함께 아이콘 및 텍스트 라벨을 병기합니다.
