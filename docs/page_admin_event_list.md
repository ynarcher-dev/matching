# [기능 명세] 관리자 - 행사 목록 및 개설 페이지 (Admin Event List)

본 문서는 관리자가 행사를 개설하고 조회, 수정 및 취소할 수 있는 행사 관리 화면의 상세 사양을 정의합니다.

---

## 1. 행사 목록 화면 (Event List View)

### 1.1 UI/UX 개요
- **진입 경로**: 관리자 로그인 후 첫 화면 (`/admin/events`).
- **주요 기능**: 개설된 전체 행사 목록을 한눈에 볼 수 있는 통합 테이블 뷰를 제공하며, 상태별 필터링과 검색, 정렬 및 페이지네이션을 지원합니다.

### 1.2 화면 구성 요소 (Layout Spec)
1. **헤더 영역**:
   - 화면 타이틀: "행사 운영 관리"
   - 버튼: "+ 새 행사 개설" (최고관리자만 노출, 클릭 시 행사 등록 및 편집 모달인 `EventFormModal` 오픈)
2. **필터 및 검색 바 (`FilterBar`)**:
   - 진행 상태 필터 칩스 (`FilterChips`): `전체` | `대기` | `예약` | `배치 조율` | `진행` | `종료` | `취소`
   - 검색창 (`SearchInput`): 행사명에 대한 실시간 디바운스 키워드 검색 지원.
3. **행사 목록 테이블 (`DataTable`)**:
   - 공통 테이블 인프라를 활용하여 누적 일람 표시.
   - 컬럼 구성:
     - `No`: 행사의 고유 번호 (기본 정렬: 생성일 기준 내림차순, 즉 최신 행사가 가장 높은 숫자를 부여받음)
     - `행사명`: 클릭 시 행사 상세 페이지로 이동
     - `상태`: 현재 행사의 진행 상태 배지 (`DRAFT`, `BOOKING`, `ALLOCATION`, `PROGRESS`, `FINISHED`, `CANCELLED`)
     - `예약 기간`: 신청 시작 ~ 종료 일시
     - `행사 기간`: 행사 시작 ~ 종료 일시
     - `참가 스타트업`: 배정된 스타트업 수 및 총 예약률 통계
     - `참가 전문가`: 배정된 전문가 수 및 테이블 설정 통계
     - `작업`: 관리자용 수정/취소 액션 버튼그룹
4. **페이지네이션 (`Pagination`)**:
   - 한 페이지당 30개씩 행사를 페이지네이션하여 조회.
5. **빈 화면 대응 (Empty State)**:
   - 조건에 맞는 행사가 없을 시 안내 메시지 표시 (권한에 따라 다른 안내 텍스트 노출).

## 2. 새 행사 개설 및 편집 화면 (Create / Edit Event Form)

### 2.1 입력 필드 구성
- **행사명**: 필수 입력 (텍스트 필드).
- **최대 상담 횟수 제한**: 스타트업당 예약할 수 있는 상담 세션 한도 수치 설정 (기본값: 3회).
- **시간대 (Timezone)**: 지역별 시간 설정 (기본값: `Asia/Seoul` - 셀렉트 박스 형태로 선택).
- **동일 전문가 중복 예약 허용 여부 (`allow_duplicate_expert`)**: 토글 스위치 (기본값: OFF). ON으로 설정 시 동일 스타트업이 한 행사 내에서 동일한 전문가와 연속 시간 등 중복 예약하는 것을 허용합니다. (단, 동시간 충돌, 테이블 충돌 및 최대 상담 횟수 제한은 여전히 유효합니다.)
- **예약 기간 설정**:
  - 예약 시작 일시 (`datetime-local`)
  - 예약 마감 일시 (`datetime-local`) - 예약 시작보다 미래 시점이어야 함.
- **행사 진행 기간 설정**:
  - 행사 시작 일시 (`datetime-local`)
  - 행사 종료 일시 (`datetime-local`) - 행사 시작보다 미래 시점이어야 함.
- *유효성 검증*: 예약 기간과 행사 기간이 정상 순서인지 확인 (예약 마감 $\le$ 행사 시작 권장).

### 2.2 등록 프로세스 및 상태 전환
- **대기(DRAFT) 상태 자동 부여**: 행사가 새로 개설되면 기본적으로 `DRAFT` 상태로 설정됩니다.
- **자동 상태 전환**:
  - Supabase Cron이 1분마다 행사 일정을 확인해 `DRAFT → BOOKING → ALLOCATION → PROGRESS → FINISHED` 순서로 상태를 전환합니다.
  - 예약 마감부터 행사 시작 전까지는 `ALLOCATION` 상태이며 스타트업 예약을 닫고 관리자 조정만 허용합니다.
- **수동 예외 처리**:
  - 일반적인 돌발 상황은 예약·행사 일정을 수정해 처리합니다.
  - 상태 직접 변경은 최고 관리자만 가능하며 사유를 필수로 입력하고 감사 로그를 남깁니다.
  - `status_override`가 설정된 동안 Cron은 상태를 덮어쓰지 않습니다.
  - 완료된 상담일지가 하나라도 있으면 `DRAFT` 또는 `BOOKING`으로 되돌릴 수 없습니다.
- **행사 취소**:
  - 행사를 물리 삭제하지 않고 `CANCELLED` 상태로 보존하며 취소 필터에서 조회합니다.

## 3. 코드 설계 가이드라인
- **컴포넌트 분할**:
  - 메인 페이지: `EventList.jsx` (목록 렌더링 담당)
  - 등록 모달/폼: `EventCreateForm.jsx` (독립 폼 컴포넌트, 유효성 검증 로직 포함)
  - 상태 배지 컴포넌트: `EventStatusBadge.jsx` (컬러 스타일 분기 처리)
- **파일 크기 제어**:
  - 날짜 처리 라이브러리(e.g. date-fns 등) 연동 유틸리티와 폼 컴포넌트를 분리하여 파일당 **500줄 이하**로 유지합니다.
- **디자인 제약**: 인라인 스타일을 배제하고 Tailwind CSS의 그리드(`grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-6`)를 적극 적용하여 반응형 카드 배치를 구현합니다.
