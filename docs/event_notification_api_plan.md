# 행사별 카카오톡/SMS 알림 API 연동 기획

> 작성일: 2026-06-26  
> 상태: Phase 7 알림 기능 재정의 초안. 기능은 구현해두되, 실제 발송은 공급사 키와 발신 설정 입력 후 활성화하는 구조를 목표로 한다.  
> 관리자 토글/활성화 게이트/완료 기준 상세는 [`page_admin_notification_settings.md`](./page_admin_notification_settings.md)를 따른다.

## 1. 목표

행사별로 알림 채널 정책을 선택하고, 카카오 알림톡/SMS 발송 API를 붙일 수 있게 한다.

핵심 방향:

- 인증 로그인과 알림 발송을 분리한다.
- 기본 무료 운영에서는 실제 발송하지 않는다.
- 기능은 구현해두고, API 키/발신번호/템플릿 설정을 넣으면 활성화할 수 있게 한다.
- 행사별로 알림 채널 정책을 선택한다.

## 2. 행사별 채널 정책

행사 설정에서 다음 중 하나를 선택할 수 있게 한다.

| 정책 | 의미 |
|---|---|
| 발송 안 함 | 무료 운영 모드. 알림 로그도 실제 발송하지 않음 |
| 카카오 알림톡만 | 알림톡 API만 사용 |
| SMS만 | SMS API만 사용 |
| 카카오 알림톡 + SMS | 알림톡 우선, 실패 또는 미수신 시 SMS fallback |

권장 기본값은 `발송 안 함`이다.

## 3. 설정 범위

전역 설정:

- 공급사 선택
- API Key
- API Secret
- 발신번호
- dispatch secret
- 실제 발송 활성화 여부

행사별 설정:

- 알림 채널 정책
- 사용할 템플릿 묶음
- 예약 시작 안내 발송 여부
- 예약 변경/취소 안내 발송 여부
- 미예약 리마인드 발송 여부
- 행사 전 리마인드 발송 여부

## 4. 발송 이벤트

우선 구현 대상:

- BOOKING 전환 시 예약 시작 안내
- 예약 생성
- 예약 변경
- 예약 취소
- 미예약 스타트업 리마인드
- 행사 전 리마인드

추후 검토:

- 상담 시작 전 리마인드
- 만족도 조사 요청
- 운영자 수동 공지

## 5. 기존 구현과 연결

현재 알림 인프라는 다음 방향으로 이미 일부 준비되어 있다.

- `notification_logs`
- 알림 중복 방지 키
- PENDING/SENT/FAILED 상태
- 재시도와 백오프
- `notification-dispatch` Edge Function
- 관리자 알림 현황 탭

후속 작업은 이 인프라에 실제 공급사 어댑터와 행사별 정책을 연결하는 것이다.

## 6. 공급사 어댑터

1차 공급사는 Solapi를 우선 검토한다.

필요 어댑터:

- MockNotifier
- SolapiAlimtalkNotifier
- SolapiSmsNotifier
- CompositeNotifier 또는 FallbackNotifier

Fallback 정책:

```text
카카오 알림톡 + SMS 정책
1. 알림톡 발송 시도
2. 공급사 실패 또는 fallback 대상이면 SMS 발송 로그 생성
3. SMS 발송 시도
4. 각각의 결과를 notification_logs에 남김
```

## 7. 템플릿 정책

카카오 알림톡은 템플릿 사전 등록/승인이 필요하다.

따라서 시스템은 템플릿 본문을 자유롭게 보내기보다, 승인된 템플릿 코드와 치환 변수를 저장하는 구조를 우선한다.

예상 템플릿:

- 예약 시작 안내
- 예약 확정 안내
- 예약 변경 안내
- 예약 취소 안내
- 미예약 리마인드
- 행사 전 리마인드

## 8. 관리자 UI

행사 설정 또는 행사 상세에 다음 UI를 추가한다.

- 알림 발송 정책 선택
- 템플릿 선택
- 발송 활성화 상태 표시
- 테스트 발송 버튼
- 발송 로그/실패 로그
- 실패 건 수동 재시도

API 키는 일반 행사 관리 화면에 직접 노출하지 않는다. 전역 환경변수 또는 관리자 전용 설정으로 다룬다.

상세 구현 기준:

- 관리자 페이지의 전역 `실제 발송 활성화` 토글 기본값은 OFF다.
- 행사별 정책 기본값은 `발송 안 함`이다.
- 전역 토글 OFF, 행사 정책 `발송 안 함`, 이벤트별 토글 OFF, 공급사 설정 불완전 상태에서는 외부 API를 호출하지 않는다.
- 작업 전 [`관리자 알림 설정 및 발송 활성화 명세`](./page_admin_notification_settings.md)를 반드시 읽는다.

## 9. 비용 메모

Solapi 공개 가격표 기준:

| 채널 | 단가 |
|---|---:|
| 카카오 알림톡 | 13원/건, VAT 별도 |
| SMS | 18원/건, VAT 별도 |
| LMS | 45원/건, VAT 별도 |
| MMS | 110원/건, VAT 별도 |

비용만 보면 카카오 알림톡이 SMS보다 저렴하다. 다만 템플릿 승인과 실패 fallback 운영을 고려해야 한다.

## 10. 미결정 사항

- 실제 1차 공급사를 Solapi로 확정할지
- API 키를 Supabase Edge Function 환경변수로만 둘지, 관리자 설정 화면을 만들지
- 행사별 발신번호가 필요한지
- SMS fallback을 자동으로 할지, 운영자가 수동 재시도하게 할지
- 무료 운영 모드에서 PENDING 로그 자체를 만들지, 로그만 만들고 발송을 건너뛸지
