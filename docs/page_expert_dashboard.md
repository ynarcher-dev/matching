# [기능 명세] 전문가 - 상담 현황 및 디지털 상담일지 (Expert Dashboard & Logs)

본 문서는 전문가(자문/멘토)가 로그인하여 자신의 세션을 조회하고 상담 내용을 기록하는 전문가 전용 화면의 상세 사양을 정의합니다.

---

## 1. 전문가 메인 대시보드 화면 (Expert Dashboard)

### 1.1 UI/UX 개요
- **진입 경로**: 전문가 계정으로 로그인 시 즉시 진입 (`/expert/dashboard`).
- **목적**: 모바일 현장 가독성을 극대화하여 오늘 진행해야 할 시간대별 스타트업 미팅 정보와 상태를 실시간으로 조작 및 감시하는 화면입니다.

### 1.2 화면 구성 요소 (Mobile Spec)
1. **전문가 프로필 카드 및 행사 스위처 (Top Section)**:
   - 전문가 성명, 소속, 등록 분야 표기.
   - 복수의 참가 행사가 배정된 경우, 상단 탭 컴포넌트(`Tabs`)를 통해 행사를 간편하게 전환(`setSelectedId`)할 수 있습니다.
2. **진행 중인 세션 강조 카드 (`ActiveSessionCard`)**:
   - 현재 시간대에 예정되어 있거나 진행 중인 스타트업 상담 매칭을 상단에 대형 카드로 우선 노출합니다.
   - 제공 정보: 스타트업명, 대표자명, 기업 요약 소개글, 첨부된 사업소개서(PDF) Signed URL 링크, 실제 진행 상담 테이블 위치.
   - **카운트다운 타이머 (`CountdownTimer`)**: 현재 세션의 남은 시간(`MM:SS`)을 1초 주기로 연산하여 실시간 노출하며, 남은 시간이 5분 미만일 경우 적색 경고 점멸을 통해 진행 경고를 제공합니다.
   - 조작 컨트롤 (`ExpertAttendanceControl`):
     - 본인 및 상대 기업의 출석을 3단 컨트롤로 각각 선택합니다.
     - `상담 시작` 버튼: 세션이 `WAITING` 단계일 때만 노출되며, 클릭 시 상담을 시작하여 세션을 `IN_PROGRESS`로 변경합니다.
     - `상담일지 작성`: 일지 작성 모달(`CounselingLogModal`)을 활성화합니다.
3. **전체 미팅 일정 리스트 (`ExpertScheduleList`)**:
   - 해당 전문가에게 배정된 금일 전체 시간대 매칭 슬롯 목록.
   - 상태 배지 적용: `대기`, `진행 중`, `완료`, `노쇼`, `취소` 상태에 따라 색상과 아이콘 및 텍스트 라벨을 제공합니다.
   - 예약되지 않은 시간대는 "예약 없음"으로 표기됩니다.

## 2. 디지털 상담일지 및 스코어카드 작성 화면 (Counseling Log Form)

### 2.1 스코어카드 입력란
전문가가 스타트업을 다차원적으로 평가하기 위한 5점 척도(1 ~ 5점) 선택 버튼 또는 슬라이더 UI를 제공합니다.
- **평가 항목**:
  1. `기술성 (Technology)`: 보유 기술의 깊이 및 구현 완성도.
  2. `전문성 (Expertise)`: 팀의 구성 및 전문 분야 역량.
  3. `신뢰도 (Reliability)`: 인터뷰 태도 및 커뮤니케이션 성실함.
  4. `협업 잠재력 (Collaboration Potential)`: 파트너십 또는 후속 연계 가능성.
  5. `거래 가능성 (Probability)`: 단기 내 실질적 비즈니스 매칭 성사 확률.

### 2.2 상담 내용 입력 (Free Text)
- **상담 의견란**: 스타트업의 애로사항 및 상담 코칭 요약본을 기록하는 텍스트 영역. 최소 글자 수 제한이 없으며 임시저장이 가능합니다. (행사 종료 전까지 수정 및 제출 필요)
- **후속 연계 요청**: 향후 추가 매칭 필요성 체크박스 및 메모란.
- **공개 여부**: 점수는 항상 관리자 전용이며, 텍스트 코멘트에 대해서만 스타트업 공개 여부를 선택.

### 2.3 세션 종료 처리 (Session End Trigger)
- 전문가가 "상담 완료 및 제출" 버튼을 터치할 시:
  1. 5개 필수 점수를 검증합니다. 코멘트는 미입력 시 임시저장 상태로 둘 수 있으며, 최종 제출 전까지 완성이 필요합니다.
  2. `counseling_logs` 저장과 `matching_slots.session_status = 'COMPLETED'` 전환을 하나의 DB RPC 트랜잭션으로 처리합니다.
  3. 대시보드 화면으로 즉시 복귀하며 다음 세션 대기 화면으로 갱신됩니다.
  4. 상담 결과 공개 알림은 공개 정책과 행사 종료 시점에 맞춰 별도 비동기 처리합니다.
- 상담일지는 행사 `FINISHED` 전까지 수정할 수 있으며 수정 이력을 감사 로그에 남깁니다.
- 미진행 세션은 관리자 권한으로 `NO_SHOW` 또는 `CANCELLED` 처리합니다.

### 2.4 출석 처리 및 노쇼
- 전문가는 대시보드 진행 세션 영역(`ExpertAttendanceControl`)에서 본인 및 스타트업에 대해 각각 출석 체크를 직접 수행할 수 있습니다.
- 컨트롤은 `[미정 | 출석 | 불참]` 3단 선택지로 구성되며, 실수로 체크한 경우 미정을 다시 선택해 복구할 수 있습니다.
- *보류*: 30초 유효시간 서명용 QR 스캔 출석 체크 기능은 현장 단순화를 위해 취소 및 보류되었습니다.

### 2.5 행사 만족도 조사
- 행사가 `FINISHED` 상태가 되면 대시보드 시간표 영역에 동적 만족도 설문 조사 패널(`SatisfactionPanel`)이 노출됩니다.
- 문항은 관리자가 행사 상세에서 정의한 동적 폼 렌더러 기반으로 동작하며, 1회만 제출 가능하고 수정할 수 없습니다.

### 2.6 이전 상담 이력 조회
- 별도의 메뉴를 통해 전문가가 진행했던 과거 완료(`COMPLETED`) 세션의 상담일지 이력(`/expert/history` -> `ExpertHistoryView.tsx`)을 통합 조회할 수 있습니다.
- 이 화면에서는 이전 일지 요약 내용(`CounselingLogSummary`)을 제공하며 읽기 전용 상태로 안전하게 보존합니다.

## 3. 코드 설계 가이드라인
- **컴포넌트 분할**:
  - 메인 뷰: `ExpertDashboardView.tsx` (행사 스위칭 및 활성 세션 카드 배치)
  - 이력 뷰: `ExpertHistoryView.tsx` (과거 완료 상담 목록 일괄 조회)
  - 활성 슬롯 타이머: `CountdownTimer.tsx` (1초 단위 카운트다운 배지)
  - 일지 작성 모달: `CounselingLogModal` (다차원 평가 점수 및 텍스트 폼)
  - 만족도 설문: `SatisfactionPanel.tsx` (참가자 공용 동적 설문 제출 폼)
  - 스케줄 리스트: `ExpertScheduleList.tsx`
- **최대 라인 제한**:
  - 카운트다운 타이머 포맷 연산 및 일정 분류 헬퍼 함수는 `lib/expertSchedule.ts`에, 상담 평가 유효성 검증은 `lib/counseling.ts`에 분리하여 파일당 **500줄 규정**을 충족합니다.
- **디자인 제약**: 모바일 터치 조작의 실수를 줄이기 위해 평가용 라디오 버튼 및 제출 단추의 크기를 충분히 크게(`py-4 px-6 text-lg`) 조절하고 Tailwind CSS 모바일 가이드를 준수합니다.
