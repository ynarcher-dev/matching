# 개발 현황 및 마일스톤 체크리스트

이 문서는 현재 코드베이스를 기준으로 구현 상태를 요약한 최신 개발 현황 문서입니다.
세부 기획과 작업 이력은 각 페이지 문서와 `worklog_*.md` 문서를 참고합니다.

## 전체 진행률

`[########################] 97%`

2026-07-01 기준, 오프라인 비즈니스 매칭 행사 운영에 필요한 핵심 흐름은 대부분 구현되어 있습니다. 남은 작업은 실 알림 사업자 연동, 운영 환경 Cron/Vault 설정, 주요 뷰포트 및 실제 기기 검증, 일부 보조 운영 화면 보강이 중심입니다.

## 현재 기술 구성

- Frontend: Vite + React + TypeScript + Tailwind CSS v4
- State/Data: Zustand, TanStack Query, React Hook Form, Zod
- Backend: Supabase DB/RLS/RPC/Storage/Edge Functions
- 인증:
  - 운영자: Supabase Auth + `users.auth_user_id`
  - 참가자: 이름 + 휴대전화 기반 무료 로그인, 참가자 전용 JWT
  - 긴급 1회용 로그인 링크: `/login/emergency`
- DB 마이그레이션: `0001_schema.sql`부터 `0075_reopen_counseling_log.sql`까지 존재
- 검증 체계: `lint`, `typecheck`, `build`, `vitest`, 운영자 권한 검증 스크립트

## 주요 구현 영역

### 인증과 공통 레이아웃

- 운영자/관리자/현장스태프/전문가/스타트업 역할 기반 라우팅
- `RequireAuth`, `RequireRole`, `RequireSuperAdmin` 가드
- `AppShell`, `Header`, `Sidebar` 기반 데스크톱/모바일 공통 레이아웃
- 참가자 OTP 흐름 제거 후 이름+휴대전화 로그인으로 전환
- 참가자 세션 버전과 무효화 흐름 유지

### 관리자 백오피스

- 행사 목록, 생성, 수정, 취소
- 행사 상세:
  - 스타트업/전문가 배정
  - 테이블 생성 및 테이블 담당자 지정
  - 슬롯 생성, 예약 현황, 진행 관리
  - 운영 관리 모달, 결과/설정 패널
- 참가자 DB:
  - 스타트업/전문가 분리 관리
  - 검색/필터, 개별 추가/수정
  - CSV 일괄 업로드
  - 분야, 파일, 프로필 정보 관리
- 운영자 계정 관리:
  - 생성, 수정, 비활성화, 재활성화
  - 비밀번호 재설정
  - 행사별 OWNER/MANAGER/STAFF/VIEWER 권한 부여/회수
- 결과 내보내기:
  - 예약, 출석, 상담 결과, 만족도, 참가자 명단 시트
- 전역/행사별 알림 설정 및 로그 화면
- 행사 알림 탭은 전역 토글(`event_notification_tab_enabled`)로 노출 제어

### 스타트업 포털

- 참여 행사 목록과 행사별 예약 현황
- 전문가/시간대 기반 예약, 변경, 취소
- 예약 기간 이후에도 행사 설정에 따라 ALLOCATION/PROGRESS 단계 자기 예약 변경 허용
- 전문가 프로필, 분야, 가능 시간, 프로필 이미지 표시
- 상담 요청사항(`matching_slots.counseling_request`) 입력
- 회사 소개, 사업소개서/IR PDF 업로드
- 업로드 이력과 원본 파일명 동기화
- 참고 URL 다중 관리:
  - `company_links` 테이블
  - URL + 부가 설명 최대 20개
  - 첫 번째 URL은 기존 `users.company_homepage`와 동기화
- 행사 종료 후 스타트업/전문가 만족도 및 공개 코멘트 확인

### 전문가 대시보드

- 전문가 본인 참여 행사와 전체 상담 일정 표시
- 상담 일정에서 기업을 선택하면 Split View 상담 워크스페이스로 전환
- Split View:
  - 좌측: 사업소개서/IR PDF, 회사 정보, 상담 요청사항, 참고 URL
  - 우측: 상담일지 작성/임시저장/최종 제출
  - 데스크톱에서 좌우 비율 드래그 조절, 더블클릭 50:50 초기화
  - 1024px 미만에서는 상하 스택으로 전환
- 상담 시작 시 `start_counseling` RPC로 세션을 `IN_PROGRESS` 전환하고 전문가/스타트업 출석을 자동 처리
- 상담일지:
  - 행사별 동적 질문
  - 평점, 단답형, 장문형, 단일/복수 선택
  - 후속 연계 여부와 메모
  - 공개 여부
  - 5초 디바운스 자동 임시저장 및 수동 임시저장
  - 최종 제출 시 `COMPLETED` 전환
  - 제출 완료 후 행사 종료 전 `reopen_counseling_log_v2`로 제출 취소 및 재편집 가능
