# UI/UX Rework 09-F - Startup Booking

## 1. 목표

스타트업 예약 화면을 모바일 우선으로 개선한다. 사용자가 현재 예약 상태를 빠르게 파악하고, 예약 가능 시간을 쉽게 선택하며, 완료 후 만족도 입력을 자연스럽게 할 수 있도록 한다.

## 2. 대상

- `StartupPortalView`
- `MyBookingList`
- `BookingSlotsGrid`
- `ChangeBookingModal`
- `SatisfactionPanel`
- `ExpertSatisfactionPanel`

## 3. 현재 문제

- matrix 보기는 데스크톱에는 유효하지만 모바일에서는 가로 스크롤 의존도가 높다.
- 전문가별 보기와 시간대별 보기의 목적 차이가 UI에서 충분히 드러나지 않는다.
- 예약 가능, 마감, 내 예약, 신청 불가 상태가 색과 텍스트로 표현되지만 공통 slot button 규칙이 없다.
- 전문가 프로필/분야/소개/사진 표시가 더 강화될 수 있다.

## 4. 작업 범위

- **상단 행사 전환 탭 공통화**: `StartupPortalView.tsx`의 하드코딩된 다중 이벤트 선택 버튼(capsule형 `<button>`)들을 공용 `Tabs` 컴포넌트로 교체하여 페이지 상단 탭 규칙에 맞추어 일관성 확보.
- 모바일 기본 보기: 시간대별 보기 우선
- 전문가별 보기: 전문가 프로필 카드 중심으로 재구성
- `SlotButton` 또는 `SlotStatusBadge` 도입
- 내 예약 영역을 상단에 더 명확히 표시
- 예약 변경 모달에서 기존 예약과 변경 대상 비교 강화
- 만족도 입력 UI의 rating/options/text 패턴 정리

## 5. 보기 모드 기준

시간대별 보기:

- 빠르게 예약 가능한 시간을 찾는 목적
- 시간 그룹, 전문가 칩, 예약 가능 상태 중심
- 모바일 기본값 후보

전문가별 보기:

- 전문가 정보를 보고 선택하는 목적
- 사진, 이름, 소속, 직함, 분야, 소개, 가능 시간 표시
- 데스크톱/보조 탐색에 적합

## 6. 완료 기준

- 모바일에서 주요 예약 플로우가 가로 스크롤 없이 가능하다.
- 예약 상태가 공통 컴포넌트로 표현된다.
- 전문가 정보가 예약 선택 전에 충분히 보인다.
- 완료 후 만족도 입력 화면이 행사/전문가 만족도를 명확히 구분한다.
