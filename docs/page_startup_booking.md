# [기능 명세] 스타트업 - 예약 신청 및 취소 화면 (Startup Booking Portal)

본 문서는 스타트업 대표 및 실무자가 로그인하여 비즈니스 매칭 일정을 등록, 변경 및 취소할 수 있는 전용 예약 포탈의 상세 사양을 정의합니다.

---

## 1. 스타트업 메인 화면 (Startup Dashboard)

### 1.1 UI/UX 개요
- **진입 경로**: 스타트업 계정(등록된 이름 + 휴대전화번호) 로그인 성공 시 진입 (`/startup/booking`).
- **역할 및 주요 기능**: 
  - 모바일 터치 지향 시간표 및 스케줄러 뷰.
  - 가용 전문가 검색 및 타임 슬롯 조회.
  - 스타트업 본인의 예약 신청, 변경 및 취소 제어.
  - 복수의 비즈니스 매칭 행사에 소속된 경우, 상단 탭 스위처를 통해 행사를 유연하게 스위칭.

### 1.2 화면 구성 요소 (Mobile Spec)
1. **행사 선택 스위처 (Tabs)**:
   - 복수의 활성 행사가 있을 경우, 상단 탭 컴포넌트(`Tabs`)를 통해 행사별 일정표로 신속하게 교체 전환합니다.
2. **나의 예약 현황 (`MyBookingList`)**:
   - 현재 확정된 예약 시간대, 매치된 전문가명 및 테이블 번호, 상담 유형(수동/AI)을 카드로 표시.
   - 현황 정보 배지: `예약 완료 N회 / 최대 M회 제한` 안내.
   - 각 예약 카드 내부의 `시간 변경` (변경 모달 오픈) 및 `예약 취소` (직접 취소) 조작 제공.
3. **예약 신청 일정표 (`BookingSlotsGrid`)**:
   - Y축에 전문가 프로필 카드를 표시하고, X축(또는 모바일 확장 영역)에 가용 시간대를 나열합니다.
   - 슬롯 칩스 배지 컬러:
     - `신청 가능` (가용 슬롯): 민트색 배경
     - `마감` (타사 예약됨): 회색 배경
     - `내 예약`: 강조 컬러 표시
4. **IR/소개서 업로드 패널 (`ProposalUploadPanel`)**:
   - 스타트업이 직접 본인의 사업소개서(PDF 파일 한정)를 드래그 앤 드롭 및 파일 탐색을 통해 업로드 및 갱신합니다.
   - 소개서 업로드 시 파일명과 업로드 완료 시각이 실시간 표시됩니다.
   - 파일은 비공개 버킷에 저장되며, 권한 검증을 통과한 단기 Signed URL 형태로 관리자 및 배정 전문가에게만 다운로드 링크를 노출합니다.

### 1.3 전문가별 보기 개선 기준

전문가별 보기는 시간표를 전문가별로 나열하는 수준이 아니라, 전문가 정보를 먼저 탐색하고 해당 전문가의 가능 시간대를 선택하는 화면이어야 한다.

- 전문가 카드/행 정보: 이름, 소속, 직책, 전문 분야, 소개, 프로필 사진.
- 전문가별 가능 시간대: 예약 가능/마감/내 예약 상태를 텍스트와 버튼 상태로 구분한다.
- 같은 전문가를 이미 예약한 경우, 동일 전문가 중복 허용 정책에 따라 추가 예약 가능 여부를 명확히 표시한다.
- `시간대별 보기`는 빠른 예약 탐색용으로 유지하고, `전문가별 보기`는 전문가 정보 탐색과 선택을 우선한다.

## 2. 세션 예약, 변경 및 취소 기능 (Booking Management)

### 2.1 신규 예약 신청 로직
1. 스타트업이 빈 타임 슬롯 터치 시 예약 신청 팝업 노출.
2. 팝업 내용: `"해당 시간[{시간}]에 {전문가명} 전문가와의 상담을 예약하시겠습니까?"`
3. 유효성 검사 (클라이언트 및 서버 공동 수행):
   - **행사 상태 판별**: 신규 예약, 변경 및 취소는 `BOOKING` 상태에서만 허용.
   - **예약 가능 횟수 초과 판별**: 스타트업이 이미 신청한 총 예약 횟수 $\ge$ 행사 설정의 `max_sessions_per_startup` 수치일 경우 예약 생성을 차단하고 경고 창(`"예약 한도 초과: 행사별 최대 M회까지만 예약할 수 있습니다."`) 출력.
   - **스케줄 중복 점검**: 해당 시간대에 스타트업의 다른 상담 슬롯(또는 타 행사의 동시간 매칭 슬롯)이 예약 완료된 상태인지 충돌 분석 후 경고 처리.
   - **동일 전문가 중복 점검**: 한 행사에서 동일 전문가와 두 번 이상 예약할 수 없음. 단, 행사 개설/편집 시 관리자가 **"동일 전문가 2회 이상 예약 허용"(`allow_duplicate_expert`)** 토글을 ON 한 행사에서는 (연속 시간 등) 동일 전문가 복수 예약을 허용한다(동시간 충돌·테이블 충돌·최대 횟수는 여전히 적용). 기본값 OFF.
   - 최종 검증과 슬롯 점유는 DB RPC의 단일 트랜잭션으로 처리.

### 2.2 예약 일정 변경
- 기존 예약된 세션의 `시간 변경` 클릭 시:
  - 예약 변경 가능 슬롯 목록을 팝업 형태로 노출하며 별도 임시 Hold 상태는 만들지 않음.
  - 새 슬롯 선택 후 기존 슬롯 해제와 새 슬롯 예약을 하나의 DB RPC 트랜잭션으로 처리.
  - 충돌 발생 시 전체 변경을 롤백하고 기존 예약을 유지.

### 2.3 예약 취소
- 예약 취소 버튼 터치 시 확인 컨펌 창 제공: `"상담 일정을 취소하시겠습니까? 취소된 시간은 다른 기업이 즉시 예약할 수 있게 됩니다."`
- 확인 클릭 시 DB 매칭 슬롯 레코드의 `startup_id`를 NULL로 업데이트하고 `booking_type`을 초기화합니다.
- 취소는 DB RPC로 처리하고 `booking_history`에 보존합니다. 스타트업의 취소 사유 입력은 선택 사항입니다.
- 취소된 슬롯은 즉시 다른 기업에 공개됩니다.
- 전문가와 스타트업 양방향으로 취소 안내 알림톡을 비동기 발송합니다. 알림 실패는 취소 트랜잭션을 롤백하지 않습니다.

### 2.4 운영 단계별 권한
- 스타트업은 `BOOKING` 마감 전까지 횟수 제한 없이 예약 변경과 취소를 할 수 있습니다.
- `ALLOCATION`, `PROGRESS` 단계에서는 기본적으로 직접 조작이 차단되지만, 관리자가 대시보드 상에서 `"스타트업 자율 예약 허용"` 옵션을 ON으로 설정한 경우에 한하여 예외적으로 예약 변경 및 취소가 가능합니다. 그 외의 경우에는 관리자 강제 변경만 작동합니다.
- 관리자 강제 변경이 발생하면 관련 전문가와 스타트업에 비동기 알림을 발송합니다.

### 2.5 만족도 조사 및 공개 피드백
- **만족도 조사 패널 (`SatisfactionPanel`)**:
  - 행사가 `FINISHED` 단계로 들어가면, 예약 일정표 대신 관리자가 구성한 동적 문항의 설문 폼이 활성화됩니다.
  - 설문 제출(`submit_survey` RPC 호출)은 행사당 1회만 허용되며, 제출 후 수정할 수 없습니다. 중복 제출 시 `23505` 고유 제약조건 오류를 클라이언트에서 감지하여 안전하게 대처합니다.
- **공개 피드백 패널 (`PublicCommentsPanel`)**:
  - 행사가 종료되면 전문가가 본인의 상담일지 제출 시 스타트업 공개를 승인한(`is_public = true`) 텍스트 의견 코멘트들을 확인할 수 있습니다.
  - 상담별 평가 점수 및 비공개 처리된 상담 내용은 제외되며, 보안 등급 검증(`list_public_comments` SECURITY DEFINER RPC)을 거쳐 본인에게 해당하는 세션의 공개된 내용만 필터링하여 안전하게 표출됩니다.

## 3. 코드 설계 가이드라인
- **컴포넌트 분할**:
  - 메인 포털 뷰: `StartupPortalView.tsx` (행사 스위칭 및 분기 렌더링 총괄)
  - 일정 그리드: `BookingSlotsGrid.tsx` (전문가 정보 칩스 렌더링 및 가용 타임 매트릭스 설계)
  - 나의 예약 목록: `MyBookingList.jsx`
  - IR 소개서 업로드: `ProposalUploadPanel.tsx` (PDF 단기 Signed URL 업로드 락 연동)
  - 만족도 조사: `SatisfactionPanel.tsx` (동적 설문조사 공용 컴포넌트)
  - 공개 피드백: `PublicCommentsPanel.tsx` (전문가 상담 정성 의견 리스트)
- **최대 라인 제한**:
  - 복잡한 예약 및 일정 필터링 헬퍼 함수는 `lib/startupBooking.ts`로, API 쿼리 및 뮤테이션 연동 로직은 `hooks/useStartupPortal.ts`로 분산하여 파일당 **500줄 이하**를 유지합니다.
- **디자인 제약**: 인라인 스타일 작성을 전면 배제하며, Tailwind CSS 반응형 유틸리티(`grid grid-cols-2 md:grid-cols-4 gap-4`)를 적용하여 모바일 기기 화면에서도 미팅 시간표가 깨지지 않고 축약 표기되도록 최적화합니다.
