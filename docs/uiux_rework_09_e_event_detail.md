# UI/UX Rework 09-E - Event Detail Dashboard

## 1. 목표

행사 상세 화면의 정보 구조와 기능성 UI를 재정리한다. 행사 상세는 전체 서비스에서 가장 많은 기능이 모인 화면이므로 9번째 작업의 핵심 적용 대상이다.

## 2. 현재 구성

- 행사 헤더
- 권한 배지
- 자율 예약 허용 Toggle
- 탭: 테이블 세팅, 참가 스타트업, 참가 전문가, 예약/강제조정, 진행 현황, 상담일지, 행사 만족도, 전문가 만족도, 증빙사진, 행사알림
- 설정 모달: 상담일지 설정, 행사 만족도 설정, 전문가 만족도 설정, 행사 알림 설정
- 운영자 배정 모달

## 3. 현재 문제

- 탭 수가 많아 스캔이 어렵다.
- 설정과 결과 화면의 관계가 더 명확해야 한다.
- 탭 버튼이 직접 구현되어 공통 탭 규칙과 분리되어 있다.
- 참가자 지정, 결과 목록, 사진 상태, 알림 로그의 테이블/필터 규칙이 다르다.

## 4. 작업 범위

- `PageHeader`로 행사명, 상태, 권한, 기간, 주요 액션 정리
- 상세 탭을 공통 `Tabs`로 전환
- **참가자 지정 서브탭 공통화**: `ParticipantAssignPanel.tsx`의 스타트업/전문가 서브탭 전환용 직접 구현된 버튼을 공용 `Tabs` 컴포넌트로 마이그레이션. 공용 `Tabs`가 지원하는 `count` 배지 기능을 활용해 일관되게 표시.
- **주요 링크 버튼 스타일 통일**: `AllocationToolbar.tsx` 내의 "돌아가기" `<Link>` 태그 스타일을 공용 `Button` 컴포넌트의 `outline` 및 `sm` 규격(`border-border bg-surface-raised` 등)과 일치하도록 클래스 정돈.
- 모바일에서는 탭 overflow 또는 select fallback 검토
- 설정 버튼 위치와 라벨 통일
- 결과 패널은 `ResultPanel` 패턴으로 통일
- 진행 현황 출석 세그먼트는 `AttendanceSegmentedControl`로 정리
- 사진/알림/상담/만족도 결과는 목록형 규칙 반영

## 5. 탭 재구성 방향

기본 순서:

1. 테이블 세팅
2. 참가 스타트업
3. 참가 전문가
4. 예약/강제조정
5. 진행 현황
6. 상담일지
7. 행사 만족도
8. 전문가 만족도
9. 증빙사진
10. 행사알림

개편 포인트:

- 현재 진행 상태에 맞는 기본 탭 진입은 유지한다.
- 권한이 없는 탭은 숨김 유지.
- 설정성 액션은 탭 본문 우측 상단의 공통 `SettingsButton` 사용.
- 결과성 탭은 검색/필터/내보내기 위치를 통일한다.

## 6. 완료 기준

- 탭 UI가 공통 컴포넌트로 바뀐다.
- 설정 모달 진입 방식이 탭마다 동일하다.
- 진행 현황과 결과 화면의 상태 badge/tone이 공통화된다.
- 권한별 표시/비활성 상태가 명확하다.
