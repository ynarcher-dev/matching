# 전문가 대시보드 Split View 기반 상담일지 작성 UI/UX 개선안

본 문서는 전문가의 상담 진행 및 일지 작성 편의성을 극대화하기 위해, 기존의 모달(`CounselingLogModal`) 방식과 타이머/출석 카드 중심의 UI를 개선하고, **전체 상담 일정 및 기업 IR 자료/소개서 열람과 일지 작성을 한 화면에서 동시에 수행할 수 있는 Split View(분할 화면) 기반의 전문가 전용 작업 환경**을 제안합니다.

---

## 1. 개선 배경 및 목적

### 기존 방식의 문제점 (UX Pain Points)
* **정보 열람과 일지 작성의 분리**: 전문가가 스타트업의 소개서나 IR 자료(PDF)를 확인하면서 동시에 상담 일지를 작성하려면, 모달 창을 켜고 끄거나 브라우저 탭을 여러 개 띄워 번갈아 확인해야 합니다.
* **불필요한 UI 요소에 의한 시선 분산**: 현재 대시보드 상단에 배치된 실시간 카운트다운, 전문가/스타트업 출석 마킹 버튼 등은 전문가가 실제 대시보드를 보며 상담을 진행하는 핵심 목적(상담 일정 확인 및 일지 작성)에 비해 화면 면적을 너무 많이 차지하고 복잡해 보입니다.
* **모바일/태블릿 터치 오동작**: 현장 태블릿 환경에서 복잡한 상태 값과 출석 컨트롤이 밀집되어 있어 터치 오동작의 우려가 있습니다.

### 개선 방향 (Core Concept)
> [!IMPORTANT]
> **"상담 일정 중심의 몰입형 Split View 워크스페이스 구축"**
> 1. 복잡한 타이머 및 수동 출석 체크 영역을 걷어내고 UI를 간소화합니다.
> 2. 전체 상담 일정에서 **기업을 클릭하면 즉시 화면이 Split(분할) 상태로 전환**됩니다.
> 3. 좌측 영역에서는 **기업 정보(IR자료, 참고 URL, 상담 희망사항)를 탭 구조로 즉시 확인**하고, 우측 영역에서는 **상담 일지를 바로 작성**할 수 있도록 합니다.
> 4. 사용자의 기기 환경과 선호도에 맞춰 **좌우 분할 비율을 자유롭게 드래그(Resizable Splitter)**할 수 있게 만듭니다.

## 2. 화면 구조 및 레이아웃 제안

### A. 기본 대시보드 뷰 (상담 시작 전 / 슬롯 미선택 상태)
대시보드에 진입하면 기존의 복잡한 카드를 배제하고, 깔끔하게 정리된 프로필 배너와 행사 정보, 그리고 **전체 상담 일정 목록**이 중앙에 배치됩니다.

```
+-----------------------------------------------------------------------------------+
| [비즈니스 매칭]                                         전문가/자문위원 김도현 | 로그아웃 |
+-----------------------------------------------------------------------------------+
|  김도현 책임연구원 (오늘의 상담 일정을 확인하고 진행하세요)                             |
|  행사: 2026 와이엔아처 스타트업 매칭데이 (회의 데모 LIVE)                               |
+-----------------------------------------------------------------------------------+
|                                                                                   |
|  [ 전체 상담 일정 ]                                                                |
|  +-------+--------------------+------------+----------+------------+-----------+  |
|  | 시간  | 기업명             | 매칭테이블 | 진행상태 | 출석(전/스)| 일지 작성 |  |
|  +-------+--------------------+------------+----------+------------+-----------+  |
|  | 14:38 | (스타트업)         | E-01       | 취소     |  전✓ / 스✓ | [일지보기]|  |
|  | 14:38 | 예약 없음          | -          | -        |  -         | -         |  |

|  +-------+--------------------+------------+----------+------------+-----------+  |
|                                                                                   |
+-----------------------------------------------------------------------------------+
```

### B. Split View 전환 뷰 (일정에서 특정 스타트업 클릭 시)
특정 기업의 `[일지 작성]` 또는 `기업 열람` 시 화면이 동적으로 분할되어, 하나의 캔버스에서 작업이 이루어집니다.

```
+-----------------------------------------------------------------------------------+
| [비즈니스 매칭]                                         전문가/자문위원 김도현 | 로그아웃 |
+-----------------------------------------------------------------------------------+
|  [◀ 전체 일정으로 돌아가기] | 상담 대상: 페이로직 (15:28 ~ 16:08, E-01 테이블)       |
+-----------------------------------+-----------------------------------------------+
| (1/3 영역) 상담 일정 및 퀵 네비      | (2/3 영역) 리사이저블 스플릿 패널 (Split View)  |
|                                   |  * [◀] 드래그하여 좌우 비율 조절 [▶]          |

| | 14:38 (스타트업) [완료]        | | | [좌측: 기업 정보]   | [우측: 상담일지 작성] |

| |                               | | |   |  Page 1 /  |   |                     | |
| |                               | | |   |  Introduce |   |                     | |
```

## 3. 핵심 기능 정의

### ① Resizable Splitter (드래그식 경계선)
* **동작**: PDF 뷰어와 상담일지 입력 폼 사이의 세로 경계선(Splitter)을 마우스 드래그 또는 터치 스와이프로 좌우 이동하여 너비 비율을 조절할 수 있습니다.
* **최소/최대 제한**: 어느 한쪽이 완전히 가려지지 않도록 최소 너비(예: 300px 또는 25%)를 보장하여 레이아웃이 깨지는 것을 방지합니다.
* **더블 클릭**: Splitter를 더블 클릭하면 정확히 `50:50` 비율로 자동 초기화됩니다.

### ② 기업 정보 및 자료 뷰어 (Left Pane)
전문가가 기업을 다각도로 파악할 수 있도록 **3가지 탭 구조**로 정보를 렌더링합니다.

* **[자료] 탭 (PDF/IR 뷰어)**:
  - `users.proposal_file_url`을 기반으로 Firebase Storage 등에서 Signed URL을 발급받아 내장합니다.
  - `<iframe src={signedUrl} />` 또는 PDF.js 방식을 사용하여 별도의 팝업 없이 웹 페이지 내에서 바로 스크롤 및 확대/축소하며 볼 수 있도록 지원합니다.
* **[링크] 탭 (소개 및 참고 URL)**:
  - 스타트업이 가입 또는 신청 시 작성한 **참고 URL(홈페이지, 노션 페이지, 웹 IR 등)**을 외부 링크 버튼 및 iframe 미니 프리뷰 형태로 지원합니다.
  - **기업 업로드 화면 추가**: 스타트업 마이페이지/신청서 입력 시 'IR/소개서 파일 업로드' 영역 옆에 **'참고 URL (소개 홈페이지, 웹 IR 등)'** 입력 칸을 추가적으로 배치합니다. (DB: `users.company_homepage` 컬럼 연동)
* **[요청] 탭 (상담 희망사항 및 기업 요약)**:
  - **상담 희망사항**: 스타트업이 상담 예약 또는 신청 단계에서 입력한 **'상담 희망사항 (간단한 고민거리, 자문받고 싶은 핵심 질문 등)'**을 간단한 텍스트 카드로 노출하여 전문가가 상담 시작 전에 선제적으로 인지하도록 돕습니다. (DB: `matching_slots.counseling_request` 컬럼 연동)
  - **기업 요약**: `users.company_description`에 해당하는 간단한 회사 및 비즈니스 모델 요약 텍스트를 노출합니다.

### ③ 상담일지 입력 폼 (Right Pane)
* **기존 모달의 인라인화**: 기존 `CounselingLogModal`에 포함되었던 동적 만족도 평가(RATING), 객관식/주관식 문항 필드, 후속 연계 여부, 스타트업 공개 여부 토글을 그대로 폼 형태로 가져옵니다.
* **자동 임시저장 (Auto-save)**: 사용자가 입력을 멈춘 지 5초가 지나거나 다른 입력 창으로 포커스를 옮길 때 자동으로 `임시저장(save_counseling_log_draft_v2)` RPC를 백그라운드에서 호출하여 네트워크 불안정 등에 따른 데이터 유실을 방지합니다.
* **최종 제출**: `[상담 완료 및 제출]` 클릭 시 필수 항목 검증을 완료하고 상담의 상태를 `COMPLETED`로 전환합니다.

### ④ 출석 및 상태 관리 간소화
* **수동 출석 버튼 제거**: 기존의 복잡했던 출석 여부 마킹 버튼을 제거하고, 일정 테이블 우측에 간단한 상태 배지(출석 상태: 완료 `✓`, 불참 `✕`, 대기 `–`)만 보여줍니다.
* **행동 기반 자동 출석**: 
  - 전문가가 일지를 작성하기 시작하거나 `[상담 시작]`을 누르면 해당 슬롯의 세션 상태는 `진행(IN_PROGRESS)`이 되고, 전문가와 스타트업 모두 `출석(PRESENT)`으로 자동 마킹됩니다.
  - 일지 작성을 완료하여 제출하면 세션 상태는 `완료(COMPLETED)`가 됩니다.

## 4. 기대 효과 및 장점

| 구분 | 개선 전 | 개선 후 (Split View) | 기대 효과 |
| :--- | :--- | :--- | :--- |
| **작업 집중도** | 모달 창과 메인 대시보드를 수시로 스위칭 | 단일 화면에 필요한 정보(소개서)와 입력 창이 밀착 배치 | 컨텍스트 스위칭 피로도 대폭 감소 |
| **정보 접근성** | IR 자료를 보려면 별도 다운로드 및 뷰어 필요 | 화면 내 내장 PDF 뷰어 및 링크/희망사항 탭으로 즉시 검토 | 대기 시간 최소화 및 매끄러운 진행 |
| **입력 유연성** | 고정된 창 크기에서 타이핑 | 드래그로 한눈에 보고 싶은 영역을 넓힐 수 있음 | 와이드 모니터, 태블릿 등 다양한 해상도에 유연하게 대응 |
| **운영 단순성** | 출석 따로, 세션 진행 따로, 일지 따로 조작 | 상담 일정 선택 -> IR 참고 및 일지 작성 -> 완료로 단일 흐름화 | 현장 운영 인력 및 전문가의 가이드 공수 최소화 |

## 5. 피드백 및 논의 사항

> [!NOTE]
> 1. **PDF 뷰어의 브라우저 호환성**: 모바일 및 일부 태블릿 브라우저의 경우 PDF의 iframe 내장 렌더링이 원활하지 않을 수 있습니다. 이에 대응하여 모바일 웹 뷰 환경에서는 PDF 파일을 즉시 다운로드하거나 새 탭으로 여는 대체 링크(다운로드 아이콘)를 병행하여 제공할 계획입니다.
