# 기능 보완 계획 및 태스크 체크리스트

> 작성일: 2026-06-28  
> 상태: 사용자 피드백 기반 후속 기획 문서. 구현 전 개발 범위 확정 및 체크리스트 기준으로 사용한다.

## 1. 목적

현재 시스템은 핵심 매칭, 예약, 진행, 상담일지, 만족도, 알림, 사진 업로드까지 큰 흐름이 구현되어 있다. 이 문서는 운영 편의성과 데이터 관리성을 높이기 위한 후속 보완 항목을 기능 단위로 쪼개고, 각 항목의 구현 기준과 체크리스트를 정리한다.

## 2. 현재 확인된 전제

- 참가자 로그인은 한때 OTP 기반으로 구현되었으나, 2026-06-26 이후 무료 운영 모드 우선 정책에 따라 `이름 + 휴대전화번호` 로그인으로 전환되었다.
- OTP 테이블과 일부 인증 개요/긴급 링크 인프라는 보존되어 있으며, 관리자 참가자 DB 화면에도 `최근 OTP` 관련 표현이 남아 있다.
- 따라서 `DB관리의 OTP이력`은 지금 운영 의미가 맞지 않을 가능성이 높다. 삭제보다 먼저 현재 로그인 방식에서 필요한 운영 컬럼이 무엇인지 재확인하고, `최근 로그인`, `로그인 정보`, `현장 링크`, `세션 무효화` 중심으로 재정리한다.
- 만족도 조사는 동적 설문 구조가 이미 있으나, 현재 UX는 주로 행사 전체 만족도 중심이다. 후속으로 `행사 전체`, `상담 전문가별`, `둘 다` 중 행사별 수집 정책을 선택할 수 있어야 한다.
- 상담일지와 만족도 결과는 이미 별도 탭으로 존재하지만, 운영자가 원하는 구조는 결과를 메인으로 보고 설정은 우측 상단 버튼에서 모달로 여는 방식이다.
- 데이터가 누적되는 백오피스 화면은 카드 나열보다 테이블, 검색, 필터, 정렬, 페이지네이션을 기본으로 한다.

## 3. 권장 단위 태스크

### T1. 관리자 메뉴와 정보 구조 정리

- 참가자 DB 관리를 `스타트업 DB`와 `전문가 DB`로 이원화한다.
- 기존 `설정` 메뉴는 `안내발송 관리`로 명칭 변경한다.
- 전역 알림 설정과 행사별 알림 설정의 위치를 분리한다.
  - 전역: `안내발송 관리`
  - 행사별: 행사 상세의 `행사알림` 탭 안 설정 모달
- 최고관리자 전용 메뉴와 행사 운영자 메뉴 노출 기준을 함께 재검토한다.

### T2. 참가자 DB 관리 재정비

- 스타트업 DB와 전문가 DB를 별도 라우트 또는 별도 화면으로 분리한다.
- 공통 테이블 기능을 적용한다.
  - 기본 페이지 크기: 30개
  - 검색: 이름, 기업/소속, 이메일, 연락처
  - 필터: 분야, 파일 첨부 유무, 등록일, 최근 로그인/운영 상태
  - 정렬: 등록일, 이름, 기업/소속
- OTP 관련 컬럼은 현재 정책에 맞게 재검증한다.
  - 유지 후보: 최근 OTP 상태는 유료 OTP 모드 전용 또는 내부 진단용으로 숨김
  - 대체 후보: 최근 로그인, 로그인 식별정보, 세션 상태, 1회용 현장 링크 상태
- 스타트업 DB에는 IR/소개서 업로드 상태를 핵심 컬럼으로 노출한다.

### T3. 공통 데이터 테이블 인프라

- 대량 데이터형 화면에 공통 `DataTable`, `Pagination`, `FilterBar` 패턴을 적용한다.
- 모든 데이터 테이블은 한 페이지 30개를 기본값으로 한다.
- 검색/필터/페이지네이션은 장기적으로 서버 쿼리 기반으로 전환한다.
- 우선 적용 대상: 행사 목록, 스타트업 DB, 전문가 DB, 행사 상세 참가자 목록, 예약 현황/강제 조정, 상담일지 결과, 행사 만족도 결과, 전문가 만족도 결과, 증빙사진, 행사알림 로그.

### T4. 행사 생성/세팅 확장

- 행사 세팅 시 관리 계정을 함께 지정할 수 있게 한다.
- 행사 생성 플로우에서 최고관리자가 행사별 운영자를 추가하거나 이후 권한 배정 화면으로 이어지게 한다.
- 행사 기본값에 만족도 수집 정책을 포함한다.
  - 행사 전체 만족도만
  - 상담 전문가별 만족도만
  - 둘 다
  - 수집 안 함

### T5. 행사 상세 탭 재구성

행사 상세 탭은 아래 순서로 재정리한다.

1. 테이블 세팅
2. 참가 스타트업
3. 참가 전문가
4. 예약 현황 + 강제 조정
5. 진행 현황
6. 상담일지
7. 행사 만족도
8. 전문가 만족도
9. 증빙사진
10. 행사알림

운영 원칙:

- `상담일지`, `행사 만족도`, `전문가 만족도`, `행사알림`은 결과/현황을 기본 화면으로 둔다.
- 설정은 화면 우측 상단 버튼으로 열고 모달에서 수정한다.
- 상담일지 결과는 목록 행 클릭 시 상담 내용을 상세 모달 또는 드로어로 보여준다.
- 행사알림은 현황/로그를 메인으로 두고, 설정은 우측 상단 버튼에서 모달로 연다.

### T6. 만족도 조사 수집 정책 확장

- 행사별로 만족도 수집 단위를 선택한다.
- `행사 만족도`: 기존 동적 설문 구조를 사용한다.
- `전문가 만족도`: 스타트업이 실제 상담한 전문가/세션별로 응답한다.
- 둘 다 받을 경우, 스타트업 완료 화면에서는 행사 만족도와 전문가별 만족도를 구분해 보여준다.
- 관리자 결과 화면은 `행사 만족도`와 `전문가 만족도`를 별도 탭/별도 데이터셋으로 집계한다.
- 전문가별 만족도는 상담 슬롯 기준 응답 중복을 차단한다.

### T7. 스타트업 IR 직접 업로드

- 스타트업 포털에서 직접 IR/소개서 PDF를 업로드할 수 있게 한다.
- 기본 운영 정책은 스타트업 직접 업로드이며, 미업로드 기업만 관리자가 대신 업로드한다.
- 관리자 스타트업 DB와 행사 참가 스타트업 목록에서 업로드 상태를 확인한다.
- 파일 교체 이력과 마지막 업로드 주체를 기록한다.

### T8. 스타트업 예약 화면의 전문가별 보기 개선

- 현재 전문가별 보기가 시간대별 보기에 가깝다면, 전문가 중심 UX로 재구성한다.
- 전문가별 보기에서는 전문가 정보 카드를 먼저 보여준다.
  - 이름, 소속, 직책, 전문 분야, 소개, 프로필 사진
  - 상담 가능 시간대
  - 내가 이미 예약한 전문가인지 여부
- 시간대별 보기는 현재처럼 시간 기준으로 빠르게 예약 가능한 슬롯을 찾는 화면으로 유지한다.
- 모바일 기본 탭은 실제 사용성을 검토해 `시간대별` 우선, 전문가 정보 탐색은 `전문가별` 보조로 둔다.

### T9. 카드형 화면의 테이블 전환 검토

- 운영자가 나열형으로 관리하는 화면은 테이블이 더 적합하다.
- 카드형을 유지할 화면은 현장 액션 중심 화면으로 제한한다.
- 테이블 전환 우선 화면: 행사 목록, 알림 로그, 상담일지 결과, 만족도 결과, 참가자 지정 목록.

## 4. 우선순위 제안

- P0: 관리자 메뉴/DB 분리, 공통 페이지네이션 30개, 검색/필터, OTP 컬럼 재정리.
- P1: 행사 상세 탭 재구성, 결과 메인 + 설정 모달 패턴, 상담일지 상세 보기.
- P2: 만족도 수집 정책 확장, 전문가별 만족도 데이터 모델과 응답 UX.
- P3: 스타트업 IR 직접 업로드, 전문가별 예약 보기 개선.
- P4: 백오피스 카드형 목록의 테이블 전환과 디자인 정리.
