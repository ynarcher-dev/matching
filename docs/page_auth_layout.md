# [기능 명세] 인증 페이지 및 공통 레이아웃 (Auth & Common Layout)

본 문서는 사용자의 비밀번호 없는 로그인 화면과 모바일 대응을 위한 전역 웹 셸(Web Shell) 레이아웃의 상세 사양을 정의합니다.

---

## 1. 이름 및 휴대전화번호 로그인 화면 (Auth Page)

비즈니스 매칭 시스템의 기본 참가자 로그인은 외부 발송 비용을 배제하기 위해 별도의 비밀번호나 OTP 없이 **"등록된 이름 + 휴대전화번호" 정확 일치** 방식을 사용합니다.

### 1.1 UI/UX 개요
- **대상**: 관리자, 현장 스태프, 전문가, 스타트업 공통 로그인 관문.
- **특징**:
  - 참가자(스타트업, 전문가)는 등록된 이름과 휴대전화번호를 입력하여 즉시 로그인합니다.
  - 운영진(관리자, 현장 스태프)은 이메일 주소와 비밀번호(Supabase Auth)를 이용해 로그인합니다.
  - 모든 사용자에게 동일한 로그인 URL(`/login`)이 안내됩니다.

### 1.2 화면 구성 요소 (Wireframe Spec)
- **로고 영역**: 최상단에 서비스 로고 및 행사 타이틀 표시.
- **역할 선택 탭 (`RoleTabs`)**:
  - 탭 구성: `참가 스타트업` | `전문가/자문위원` | `운영진` (Zustand 상태에 따라 입력 필드와 가이드 메시지 변경).
- **입력 필드**:
  - 참가자(`STARTUP`, `EXPERT`): `이름`, `휴대전화 번호` (등록 정보 일치 필요).
  - 운영진(`OPERATOR`): `이메일 주소`, `비밀번호`.
- **로그인 버튼**:
  - 모든 필드 값이 채워졌을 때 활성화되며 클릭 시 로딩 스피너 표시.

### 1.3 로그인 로직 및 에러 처리
- **참가자 인증 처리**:
  1. 참가자가 이름과 휴대전화 번호를 입력하고 제출합니다.
  2. 서버 및 Edge Function은 입력된 이름을 공백 제거 및 소문자 정규화하고, 휴대전화 번호는 숫자만 추출하여 `public.users`의 활성 참가자 정보와 대조합니다.
  3. 정확히 **1명**의 활성 참가자가 매칭될 경우에만 참가자 전용 커스텀 JWT(유효시간 12시간, `session_version` 포함)를 발급하고 로그인 처리합니다.
  4. 검색 결과가 0명이거나 2명 이상인 모호 매칭의 경우, 보안 및 개인정보 보호를 위해 계정 존재 여부를 알 수 없도록 동일한 generic 오류 메시지를 노출합니다.
  5. 단기간 내 로그인 시도 횟수를 IP 해시 기준으로 카운트하여 레이트 리밋(예: 10분 내 20회 초과 시 429 Too Many Requests)을 적용합니다.
- **운영진 인증 처리**:
  1. Supabase Auth(`signInWithPassword`)를 거쳐 인증 토큰을 획득합니다.
  2. `public.operators` 프로필 정보를 조회하여 활성 상태와 권한을 추가로 대조하며, 비활성 계정이거나 운영진 프로필이 없을 경우 접근을 즉시 차단합니다.
- **인증 성공 시 리다이렉션 경로**:
  - `ADMIN` $\rightarrow$ `/admin/events`
  - `STAFF` $\rightarrow$ `/staff/photos` (현장 담당자 사진 업로드 화면)
  - `EXPERT` $\rightarrow$ `/expert/dashboard`
  - `STARTUP` $\rightarrow$ `/startup/booking`
- **에러 핸들링**:
  - 정보 불일치/모호 매칭 시: `"등록된 참가자 정보와 일치하지 않습니다. 이름과 휴대전화번호를 다시 확인해 주세요."`
  - 네트워크 오류 시: `"네트워크 연결 상태를 확인하고 다시 시도해 주세요."`

### 1.4 현장 예외 처리 (1회용 긴급 로그인 링크)
- 네트워크 장애, 휴대전화 분실 등으로 정식 로그인이 불가능할 경우, 최고관리자가 참가자의 본인 확인 후 짧은 만료시간(예: 10분)의 **1회용 긴급 로그인 링크**를 발급할 수 있습니다.
- **동작 흐름**:
  1. 참가자가 관리자가 전달한 `/login/emergency?token={token}` 주소로 진입합니다.
  2. 시스템은 `consume_emergency_login_token` RPC를 실행해 해당 토큰이 유효한지 확인하고 소비 처리합니다.
  3. 토큰 검증 및 소비 성공 시, 즉시 해당하는 참가자 전용 커스텀 JWT를 획득하여 로그인 상태로 전환되며 역할별 홈 경로로 리다이렉트됩니다.
  4. 해당 토큰은 즉시 만료되며 재사용할 수 없습니다. 컴포넌트 재마운트(StrictMode 등)로 인한 중복 소비 오류를 방지하기 위해 프론트엔드는 단일 처리 락(ref)을 유지합니다.

### 1.5 OTP 발급 인프라 (참고용 - 유료 알림 확장 모드 시 활성)
- SMS/알림톡 발송 기능이 활성화된 환경(유료 확장 모드)에서 2단계 인증 혹은 OTP 기반 로그인을 재활성화할 수 있도록 인프라 구조를 보존합니다.
- OTP는 숫자 6자리이며 유효시간은 5분입니다. DB에는 해시된 값과 만료시각, 시도횟수(`auth_otp_challenges`)를 기록하며, 재발송 제한 시간은 60초입니다. 최대 검증 실패 5회 시 잠금 처리됩니다.
- 로그인 세션은 `session_version`을 통해 제어되며, 최고관리자가 세션 무효화 요청 시 `session_version`이 증가하여 기존 발급된 JWT 세션은 즉시 무효화(`401 Unauthorized`) 처리됩니다.
- 공통 행사코드를 제공할 수 있으나 행사 선택용 보조값일 뿐, OTP를 대신하는 인증 수단으로 사용하지 않습니다.

## 2. 반응형 공통 레이아웃 (Common Shell Layout)

### 2.1 UI/UX 가이드라인 (Tailwind CSS 전용)
- **모바일 퍼스트**: 현장에서 스마트폰이나 태블릿으로 주로 조작하므로 모바일 뷰포트(360px ~ 768px)를 기본 설계 기준으로 삼습니다.
- **인라인 스타일 금지**: 스타일링은 100% Tailwind 클래스를 이용합니다.
- **지정 컬러 시스템**:
  - 브랜드 강조색 (Primary Accent): `#E22213` (선명한 레드) $\rightarrow$ 핵심 조작 버튼, 하이라이트 아이템에 전면 적용.
  - 베이스 무채색 (Neutral Base): `#515151` (다크 그레이) $\rightarrow$ 헤더, 사이드바 배경, 텍스트 색 및 카드 테두리에 배치.
- **고대비 가시성 (흐리멍텅한 UI 배제)**:
  - 카드·입력창·탭에 동일한 굵은 테두리를 반복하지 않습니다.
  - 일반 카드: 흰색 배경, 12~16px 모서리, 1px 중립 회색 경계선, 필요한 경우에만 매우 약한 그림자.
  - 입력창: 기본 1px 중립 경계선, 포커스 시 브랜드 레드 링 또는 경계선.
  - 역할 탭: 굵은 칸막이 대신 연회색 컨테이너 안에서 선택된 항목의 배경과 글자색을 전환하는 세그먼트 컨트롤.
  - Primary 버튼: 브랜드 레드 면을 사용하고 별도 진회색 테두리를 두지 않습니다.
  - 오류/안내: 굵은 사각 테두리 대신 연한 상태 배경, 아이콘 또는 왼쪽 강조선을 사용합니다.

### 2.2 좌측 메뉴 네비게이션 구조 (Left Navigation Layout)
- **데스크톱**: 화면 좌측에 240px 고정 영역으로 사이드바 메뉴가 상시 표출됩니다.
- **모바일/태블릿**: 화면 좌측 상단의 햄버거 메뉴를 탭하면, 사이드바가 좌측에서 우측으로 부드럽게 슬라이드인(Slide-in) 형태로 들어오는 토글 메뉴를 지원합니다.


### 2.3 상단 헤더 (Top Header)
- **디바이스 대응**: 데스크톱에서는 고정 헤더, 모바일에서는 좌측 햄버거 메뉴 포함.
- **표시 정보**:
  - 왼쪽: 햄버거 메뉴 버튼(모바일 전용), 서비스 로고.
  - 오른쪽: 현재 로그인한 사용자의 프로필 아이콘, 유저명(`예: 홍길동 대표님`), 로그아웃 버튼.
- **실시간 남은 시간 표시 (행사 진행 중인 경우)**:
  - 현재 활성화된 세션의 카운트다운 타이머가 헤더 중앙에 눈에 띄는 배지 형태로 상시 표시됨.

### 2.4 토글형 네비게이션 사이드바 (Sidebar)
- **동작**:
  - 데스크톱: 좌측 240px 너비로 고정 노출.
  - 모바일/태블릿: 햄버거 버튼 클릭 시 좌측에서 슬라이드인(Slide-in) 형태로 등장하며 뒷배경에 반투명 오버레이(Backdrop) 추가. 오버레이 또는 닫기 버튼 클릭 시 슬라이드아웃.
- **메뉴 목록 (유저 역할별 가변 제공)**:
  - **관리자**: 행사 목록(`admin/events`), 참가자 DB 관리(`admin/users`), 설정.
  - **현장 스태프**: 출석 체크(`staff/check-in`).
  - **전문가**: 오늘의 스케줄(`expert/dashboard`), 이전 상담 이력.
  - **스타트업**: 내 예약 관리(`startup/booking`), 안내 사항.

## 3. 코드 설계 가이드라인 (개발 준수 사항)
- **최대 코드 라인**: `auth.jsx`, `Layout.jsx` 등의 핵심 파일은 컴포넌트를 서브 유닛(`LoginForm.jsx`, `Header.jsx`, `Sidebar.jsx`)으로 세분화하여 각 파일당 **500줄 제한**을 철저히 준수합니다.
- **인증 세션 상태 관리**: React Context API (`AuthContext.js`) 또는 Zustand를 이용하여 전역 세션 상태를 저장하며, 페이지 전환 시마다 Route Guard를 거치게 설정합니다.
- **보안 준수**: 상세 RLS와 RPC 정책은 [인증·권한·트랜잭션 정책](./security_transactions.md)을 따릅니다.
