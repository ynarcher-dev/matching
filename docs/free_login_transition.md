# 무료 운영 전환 및 이름/전화번호 로그인 기획

> 작성일: 2026-06-26  
> 상태: **구현 완료(2026-06-26, 마이그레이션 `0035`).** 확정 사항 — 입력값=이름+휴대전화번호(행사코드 미사용) / 기존 OTP 인프라=보존(비활성) / 이름 매칭=공백 정규화+대소문자 무시.
> 구현체: Edge `participant-login` + RPC `login_participant_by_name_phone`(+`match_participant_by_name_phone`·`normalize_name`) + 프론트 `participantLoginSchema`/`authStore.loginParticipant`/`ParticipantLoginForm`. rate limit=`participant_login_attempts`(IP 해시 기준). 라이브 라운드트립 검증 완료.

## 1. 목표

외부 발송 API 비용 없이 참가자와 전문가가 로그인할 수 있도록, 현재 OTP 기반 참가자 인증을 `이름 + 휴대전화번호` 검증 방식으로 전환하는 방안을 정의한다.

핵심 방향:

- 로그인은 외부 발송 서비스에 의존하지 않는다.
- SMS/알림톡/이메일 OTP 자동 발송은 보류하거나 유료 확장 기능으로 분리한다.
- 기존 참가자 커스텀 JWT, RLS, `participantClient`, 역할별 라우팅은 최대한 재사용한다.
- 현장 예외 대응용 1회용 로그인 링크는 유지한다.

## 2. 로그인 방식

권장 플로우:

```text
1. 참가자가 이름과 휴대전화번호를 입력한다.
2. 서버는 휴대전화번호를 숫자만 남기는 방식으로 정규화한다.
3. 활성 참가자 중 이름 + 휴대전화번호가 정확히 일치하는 사용자를 찾는다.
4. 정확히 1명만 매칭되면 기존 참가자 커스텀 JWT를 발급한다.
5. 0명 또는 2명 이상 매칭되면 로그인 실패로 처리한다.
6. 실패 응답은 계정 존재 여부를 노출하지 않는 동일 문구를 사용한다.
```

권장 입력값:

- 이름
- 전체 휴대전화번호
- 필요 시 행사코드 또는 행사 선택값

전체 휴대전화번호를 우선한다. 뒤 4자리만 쓰면 동명이인과 번호 충돌 가능성이 커진다.

## 3. 보안 정책

이름 + 전화번호 로그인은 OTP보다 보안 수준이 낮다. OTP는 연락처 소유를 증명하지만, 이름 + 전화번호는 등록 정보를 알고 있음을 확인하는 방식이다.

폐쇄형 행사 운영 도구라는 전제에서 다음 보완책을 적용한다.

- 전체 휴대전화번호 입력
- 전화번호 정규화 비교
- 모호한 매칭 차단
- 로그인 실패 응답 동일화
- 로그인 시도 rate limit 유지
- 관리자 1회용 로그인 링크 유지
- 로그인 성공/실패 감사 로그 또는 최소한 최근 로그인 기록 저장 검토

## 4. 변경 범위

프론트엔드:

- 참가자 로그인 화면을 OTP 2단계에서 이름 + 휴대전화번호 단일 폼으로 변경
- OTP 요청/검증 상태, 60초 재발송 쿨다운, 6자리 코드 입력 UI 제거
- 관리자 참가자 화면의 `인증 가능 채널`, `최근 OTP 상태` 표현 재검토

백엔드:

- 이름 + 전화번호 매칭 RPC 또는 Edge Function 추가
- 기존 `participant-otp-verify`의 커스텀 JWT 발급 흐름 재사용
- 기존 OTP 테이블/함수는 즉시 삭제하지 않고 전환 기간 비활성 또는 보존 여부 결정

- `user_guide.md`
- `development_status.md`

테스트:

- 정상 로그인

- 계정 존재 여부를 노출하지 않는 generic 응답
- rate limit

## 5. 1회용 로그인 링크

현장용 1회용 로그인 링크는 무료 운영에서도 유지한다.

- 링크 발급 자체는 외부 API 비용이 없다.
- 관리자가 링크를 복사해 수동 전달하면 무료 구조다.
- SMS/알림톡/이메일 자동 발송 버튼을 붙이는 경우에만 발송 비용이 발생한다.

## 6. 미결정 사항

- 행사코드 입력을 필수로 할지 선택으로 둘지
- 기존 OTP Edge Function과 DB 테이블을 보존할지 제거할지
- 이름 매칭을 완전 일치로 할지, 공백 정규화까지만 허용할지
- 관리자 화면에서 OTP 관련 컬럼을 어떤 운영 컬럼으로 대체할지
